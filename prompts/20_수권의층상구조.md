너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 통합과학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 통합과학1
- 단원: III. 시스템과 상호작용 > 1. 지구시스템 > 지구시스템의 구성과 상호작용
- 토픽: 수권의 층상 구조
- 대상: 고등학교 1학년
- 핵심 개념: 수권의 물은 해수(97%+)와 육수(3% 미만)로 나뉜다. 해수는 깊이에 따른 수온 분포를 기준으로 혼합층, 수온 약층, 심해층으로 구분한다. 혼합층은 태양 에너지로 가열되고 바람으로 혼합되어 수온이 높고 일정하다. 수온 약층은 깊어질수록 수온이 급격히 낮아지는 안정한 층이다. 심해층은 태양 에너지가 도달하지 않아 수온 변화가 거의 없다. 각 층의 경계는 위도와 계절에 따라 다르게 나타난다.

## 씬 구성
- 화면을 좌우 분할:
  - 좌측(50%): 3D 해양 단면 시각화
  - 우측(50%): 깊이-수온 그래프 (교과서 그림 III-5, p107)
- 좌측 3D 장면:
  - 해양 단면: BoxGeometry(6, 8, 0.5) 직사각형 단면, 깊이별 색상 그라데이션
    - 혼합층(0~200m): 0x2288cc 밝은 파랑 → 수면 근처 밝게
    - 수온 약층(200~1000m): 0x1166aa → 0x0044aa 점점 짙은 파랑
    - 심해층(1000m~): 0x002244 → 0x001122 매우 짙은 남색
  - 수면: PlaneGeometry(6, 0.5), 반투명 0x88ccff, 물결 셰이더(sin 기반 Y오프셋)
  - 태양: SphereGeometry(r=0.5, emissive 0xffcc00), 상단에 배치
  - 태양 광선: 수면으로 내려오는 노란 빛줄기(CylinderGeometry, 반투명 0xffee88, opacity 0.1) 여러 개
    - 혼합층까지만 도달, 심해층에는 도달하지 않음
  - 바람 화살표: 수면 위에 수평 화살표(→) 여러 개, 0xffffff, "바람" 라벨
    - 바람에 의한 혼합 표시: 혼합층 내부에 순환 화살표(↻)
  - 물고기/해양 생물(선택적):
    - 혼합층: 작은 물고기 실루엣 3~5개
    - 심해층: 심해 생물(발광하는 점) 1~2개
  - 층 경계선: 수평 점선(DashedLineMaterial, 0xffffff) + 라벨
    - 혼합층/수온 약층 경계: ~200m
    - 수온 약층/심해층 경계: ~1000m
  - 각 층에 이름 라벨 + 특징 텍스트
- 우측 깊이-수온 그래프:
  - Canvas 2D 렌더링
  - X축: 수온 (0°C ~ 25°C)
  - Y축: 깊이 (0m ~ 4000m, 아래로 증가)
  - 수온 프로파일 곡선 (위도별):
    - 적도: 혼합층 25°C → 수온 약층 급강하 → 심해층 2~3°C
    - 중위도: 혼합층 18°C → 수온 약층 완만한 강하 → 심해층 2~3°C
    - 극지방: 혼합층 2°C → 수온 약층 거의 없음 → 심해층 1~2°C
  - 위도별 곡선 색상: 적도(빨강), 중위도(노랑), 극지방(파랑)
  - 현재 깊이(슬라이더와 연동) 빨간 수평선 표시
  - 층 구분 배경색 (좌측 3D와 동일)
- 배경: 좌측 3D 부분 상단은 하늘(0x87CEEB), 하단은 바다(0x001122)
- 조명: AmbientLight(0.3) + DirectionalLight(0.5, 태양 위치)
- 카메라: OrthographicCamera (단면도 2D 뷰 기본) + PerspectiveCamera 전환 가능

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "깊이" 슬라이더: 0 ~ 4000 (기본값 0), 단위 m, 10 단계
  - 슬라이더 이동 시:
    - 3D에서 해당 깊이에 수평선(빨간 점선) 표시
    - 우측 그래프에서 빨간 수평선 이동
    - 하단 정보 패널 업데이트
- "위도 선택" 드롭다운: 적도 / 중위도(30°N) / 극지방(60°N)
  - 선택에 따라 수온 프로파일 곡선 변경
  - 3D 단면의 색상/온도 분포도 변경
- "계절" 드롭다운: 여름 / 겨울
  - 여름: 혼합층 두꺼움(200m), 수온 높음
  - 겨울: 혼합층 얇음(100m) 또는 두꺼움(바람 강해서), 수온 낮음
  - "혼합층의 두께는 바람이 강하게 불수록 두꺼워져" (교과서 p107 말풍선)
- "태양 광선 표시" 토글: ON/OFF (기본값 ON)
- "바람 효과 표시" 토글: ON/OFF (기본값 ON)
- "초기화" 버튼

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널로 표시:

```
🌊 수권의 층상 구조
━━━━━━━━━━━━━━━━
현재 깊이:     [값] m
현재 층:       [혼합층/수온 약층/심해층]
수온:          약 [값]°C
위도:          [적도/중위도/극지방]
계절:          [여름/겨울]

층별 특징:
  혼합층 (0~200m):
    태양 에너지 가열 + 바람 혼합 → 수온 높고 일정
  수온 약층 (200~1000m):
    깊이↑ → 수온 급격히↓, 안정한 층
  심해층 (1000m~):
    태양 에너지 미도달 → 수온 거의 일정(2~3°C)

수권 구성: 해수 97%+ / 육수 3% 미만
```

  2. "혼합층의 두께가 바람의 세기에 따라 달라지는 까닭은?"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원
- 수면 물결: ShaderMaterial 또는 PlaneGeometry vertex Y 오프셋 sin(x+t)
- 깊이-수온 그래프: Canvas 2D API
- 위도별 수온 프로파일: 수학 함수로 생성 (시그모이드 + 지수 감소 조합)
- 해양 단면 색상 그라데이션: BoxGeometry의 vertexColors 또는 ShaderMaterial(uniform depth → mix(색상1, 색상2, depth))
- 수면 물결: PlaneGeometry(6, 3, 20, 20), vertex Y를 sin(x×2 + time) × 0.03으로 매 프레임 업데이트
- 태양 광선: CylinderGeometry(rTop=0.01, rBottom=0.5, h=6) × 5개, MeshBasicMaterial(0xffee88, opacity 0.08), 혼합층까지만 도달 (y > -2에서만 visible)
- 바람 화살표: ArrowHelper(방향 (1,0,0), 길이 1.0, 0xffffff) × 3개, 수면 위에 수평 배치, "바람" Sprite 라벨
- 혼합층 순환 화살표: 반원형 화살표 2개 (CatmullRomCurve3 + TubeGeometry, r=0.02), 시계방향 회전으로 혼합 표현
- 층 경계선: LineDashedMaterial(color 0xffffff, dashSize 0.2, gapSize 0.1), 혼합층/수온약층 경계(y=-2), 수온약층/심해층 경계(y=-5)
- 깊이-수온 그래프 Canvas: 별도 <canvas>, 위도 선택 시 clearRect + 재그리기
  - 수온 프로파일 함수: T(depth) = T_surface × exp(-depth/scale1) + T_deep (시그모이드 + 지수 감소 조합)
  - 적도: T_surface=25, T_deep=2, scale1=200
  - 중위도: T_surface=18, T_deep=2, scale1=150
  - 극지방: T_surface=2, T_deep=1.5, 수온약층 거의 없음
- 깊이 슬라이더 연동: 단면도에서 수평 빨간 점선(LineDashedMaterial) + 그래프에서 수평 빨간선, 매 프레임 동기 업데이트
- dispose(): 위도 전환 시 mesh를 재생성하지 않고 material.color/opacity만 변경
- 성능: requestAnimationFrame 루프, 파티클 최대 50개, 수면 vertex 업데이트 매 프레임
- 모바일: lil-gui 기본 접힘, OrbitControls enableDamping=true
- 파일명: ocean_layers_sim.html

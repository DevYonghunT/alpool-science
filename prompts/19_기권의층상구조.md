너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 통합과학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 통합과학1
- 단원: III. 시스템과 상호작용 > 1. 지구시스템 > 지구시스템의 구성과 상호작용
- 토픽: 기권의 층상 구조
- 대상: 고등학교 1학년
- 핵심 개념: 기권은 지표로부터 약 1000km까지 분포하며, 높이에 따른 기온 분포를 기준으로 대류권, 성층권, 중간권, 열권으로 구분한다. 대류권에서는 기상 현상이 일어나고, 성층권에는 20~30km 부근에 오존층이 있다. 중간권에서는 대류는 일어나지만 수증기가 없어 기상 현상이 없고, 열권은 공기가 매우 희박해 밤낮 기온 차가 크다. 전체 대기 질량의 99%가 약 32km 이내에 집중되어 있다.

## 씬 구성
- 화면을 좌우 분할:
  - 좌측(50%): 3D 지구 + 대기층 시각화
  - 우측(50%): 높이-기온 그래프 (교과서 그림 III-2, p106)
- 좌측 3D 장면:
  - 지구: SphereGeometry(r=3.0, 64, 64), 표면에 대륙/바다 절차적 텍스처
    - 바다: 0x2255aa, 대륙: 0x447744
  - 대기층: 동심원 SphereGeometry 4겹 (반투명)
    | 층 | 높이 범위 | 3D 반지름 | 색상 | opacity |
    |---|----------|----------|------|---------|
    | 대류권 | 0~12km | r=3.0~3.36 | 0x88ccff 하늘색 | 0.12 |
    | 성층권 | 12~50km | r=3.36~4.5 | 0xaaddff 연한 파랑 | 0.06 |
    | 중간권 | 50~80km | r=4.5~5.4 | 0x6688aa 짙은 파랑 | 0.04 |
    | 열권 | 80~1000km | r=5.4~33 (축소 표현 r=8.0) | 0x3344aa 남색 | 0.02 |
  - 오존층: TorusGeometry(r=3.6, tube=0.15), 0x44ff44 연초록, opacity 0.15, 성층권 내부(20~30km)에 배치, "오존층" 라벨
  - 기상 현상 효과 (대류권에서만):
    - 구름: SphereGeometry(r=0.15~0.3, opacity 0.5) 흰색, 대류권 상부에 10~15개 랜덤 배치
    - 비/눈 파티클: Points(50개, 0x88bbff), 구름 아래에서 내려옴
  - 오로라 효과 (열권):
    - 극지방(위도 ±60~80°)에 커튼 형태의 초록빛 파티클(Points 200개, 0x44ff88)
    - 상하로 흔들리는 애니메이션
  - 유성 효과 (중간권):
    - 작은 밝은 점(SphereGeometry r=0.03, emissive 0xffddaa)이 대기를 통과하며 꼬리 그림(Line, 페이드아웃)
    - 간헐적으로 생성 (5초에 1개)
- 우측 기온 그래프 (교과서 그림 III-2):
  - Canvas 2D 렌더링
  - X축: 기온 (-100°C ~ 20°C, 일부 구간은 더 높음)
  - Y축: 높이 (0 ~ 110km)
  - 기온 프로파일:
    - 대류권(0~12km): 15°C → -55°C (감소)
    - 성층권(12~50km): -55°C → -3°C (증가, 오존층 때문)
    - 중간권(50~80km): -3°C → -90°C (감소)
    - 열권(80~110km+): -90°C → 급격히 상승 (1000°C 이상)
  - 각 층 구간을 다른 배경색으로 구분 (좌측 3D와 동일 색상)
  - 층 경계에 점선 + 층 이름 라벨
  - 현재 높이(슬라이더와 연동)를 빨간 수평선으로 표시
  - 오존층 위치(20~30km)에 녹색 강조 영역
- 배경: 0x000011 (우주)
- 조명: AmbientLight(0.2) + DirectionalLight(0.6, 태양 방향) + 그림자 비활성화
- 카메라: PerspectiveCamera(FOV 55) + OrbitControls

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "높이" 슬라이더: 0 ~ 110 (기본값 0), 단위 km, 1 단계
  - 슬라이더 이동 시:
    - 3D에서 해당 높이에 수평 원반(RingGeometry, 빨간 반투명)으로 위치 표시
    - 우측 그래프에서 빨간 수평선 이동
    - 하단 정보 패널 업데이트 (층 이름, 기온, 특징)
- "대기층 표시" 토글: ON/OFF (기본값 ON) — 각 층의 반투명 구 표시
- "기상 현상 표시" 토글: ON/OFF (기본값 ON) — 구름, 비, 오로라, 유성
- "오존층 강조" 토글: ON/OFF (기본값 ON) — 오존층 녹색 링 표시
- "층별 보기" 드롭다운: 전체 / 대류권만 / 성층권만 / 중간권만 / 열권만
  - 선택된 층만 불투명하게, 나머지 페이드
- "지구 자전" 토글: ON/OFF (기본값 ON) — 느린 Y축 회전
- "초기화" 버튼

## 교육적 요소
화면 상단에 토글 가능한 "📖 학습 가이드" 패널:
- 핵심 원리: "기권은 높이에 따른 기온 분포를 기준으로 대류권, 성층권, 중간권, 열권으로 구분됩니다. 대류권에서만 기상 현상이 일어나며, 성층권의 오존층은 자외선을 차단합니다."
- 탐구 질문:
  1. "성층권에서 높이가 올라갈수록 기온이 높아지는 까닭은 무엇인가요?" (답: 오존층이 자외선 흡수)
  2. "비행기가 성층권(약 10~12km)에서 비행하는 까닭은 무엇일까요?" (답: 안정한 층, 기상 현상 없음)
  3. "오로라는 왜 열권에서 나타나나요?"
- 실생활 예시: 기상 예보(대류권), 항공 비행(성층권), 우주 정거장(열권 상부, ~400km), 유성우(중간권에서 연소)
- 교과서 연계: "미래엔 통합과학1 III편 p106~107 — 기권의 층상 구조(그림 III-2, 그림 III-3)"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원
- 대기층: SphereGeometry 4겹, side: BackSide(내부에서 봄) 또는 FrontSide(외부에서 봄) 선택
- 기온 그래프: Canvas 2D API (별도 <canvas> 요소)
- 구름/비/오로라/유성: Points + BufferGeometry
- 높이 슬라이더: 3D 수평 원반 위치 업데이트 + 그래프 수평선 동기화
- 대기층 SphereGeometry: side=BackSide (카메라가 지구 밖에서 봄), 또는 FrontSide (카메라가 지구 안에 들어가서 봄) — 외부 관찰 뷰이므로 FrontSide 사용
- 오존층 TorusGeometry: 성층권 내부(r=3.6)에 수평 배치, 지구 적도면과 평행
- 구름: SphereGeometry(r=0.15~0.3, 8, 8) × 10~15개, geometry 재사용(1개 공유), material만 clone
- 비/눈: Points(50개), position.y -= speed × dt, y < 지표면 시 리셋 (오브젝트 풀링)
- 오로라: Points(200개), 극지방 위치에 커튼 형태 분포, Y축 sin 기반 흔들림 + 색상 0x44ff88↔0x88ffcc 보간
- 유성: SphereGeometry(r=0.03) + Line(꼬리, 5개 점, opacity 감소), 5초 간격 랜덤 생성, 중간권(r=4.5~5.4) 통과 시 밝게 발광 후 소멸
- 높이 슬라이더 연동: RingGeometry(r=3.0+높이/1000×5, tube=0.01, 빨간색)를 해당 고도에 수평 배치, 매 프레임 업데이트
- 기온 그래프 Canvas: 별도 <canvas> 요소 (position absolute, right 0), requestAnimationFrame에서 슬라이더 변경 시만 재그리기 (성능 최적화)
- 층 전환 시 opacity Tween: 선택 층 opacity 0.2, 비선택 층 opacity 0.02 (lerp 0.5초)
- dispose(): 층별 보기 전환 시 mesh를 scene.remove()하지 않고 visible=false로 토글 (dispose 불필요, 재생성 방지)
- 성능: requestAnimationFrame 루프, 파티클 최대 300개, 구름 mesh 최대 15개
- 모바일: lil-gui 기본 접힘 상태, OrbitControls enableDamping=true
- 파일명: atmosphere_layers_sim.html

너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 통합과학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 통합과학1
- 단원: III. 시스템과 상호작용 > 2. 역학 시스템 > 중력의 작용
- 토픽: 자유 낙하 운동
- 대상: 고등학교 1학년
- 핵심 개념: 지표면에서 물체가 중력만을 받아 아래로 떨어지는 운동을 자유 낙하 운동이라고 한다. 자유 낙하 하는 물체의 속력은 질량에 관계없이 1초마다 약 9.8m/s씩 일정하게 증가한다. 중력 가속도의 크기는 질량에 관계없이 약 9.8m/s²로 일정하다. 같은 높이에서 가만히 놓아 자유 낙하 하는 모든 물체는 질량에 관계없이 지표면에 동시에 도달한다.

## 씬 구성
- 화면을 좌우 분할:
  - 좌측(60%): 3D 낙하 실험 장면
  - 우측(40%): 속력-시간 그래프 + 정보 패널
- 좌측 3D 장면:
  - 배경: 연한 하늘 그라데이션 (0x87CEEB → 0xE0F0FF)
  - 바닥: PlaneGeometry(15, 15), MeshStandardMaterial(0x888888), GridHelper(15, 30) 위에
  - 높이 눈금: Y축 좌측에 세로 눈금 막대(BoxGeometry 가느다란 기둥) + 높이 라벨 (0m, 10m, 20m, ...)
  - 낙하 물체 3개 (질량 다름, 나란히 배치):
    | 물체 | 형상 | 크기 | 색상 | 질량 |
    |------|------|------|------|------|
    | 공 A | SphereGeometry(r=0.3, 32, 32) | r=0.3 | 0xff4444 빨강 | 1 kg | MeshStandardMaterial(roughness 0.4, metalness 0.2) |
    | 공 B | SphereGeometry(r=0.5, 32, 32) | r=0.5 | 0x4488ff 파랑 | 5 kg | MeshStandardMaterial(roughness 0.4, metalness 0.2) |
    | 공 C | SphereGeometry(r=0.7, 32, 32) | r=0.7 | 0x44aa44 초록 | 10 kg | MeshStandardMaterial(roughness 0.4, metalness 0.2) |
  - 공 크기가 질량에 비례하여 다름 → "질량이 달라도 동시 도달" 시각적 강조
  - 각 공 위에 질량 라벨: Sprite 또는 CSS2DRenderer, "1 kg", "5 kg", "10 kg"
  - 각 공 옆에 실시간 속력 라벨: "v = [값] m/s" (매 프레임 업데이트, 색상은 공과 동일)
  - 낙하 시 이전 위치를 반투명 잔상(ghost)으로 표시:
    - 0.1초 간격으로 SphereGeometry(동일 geometry 공유) + MeshBasicMaterial(공 색상, opacity 0.12) 생성
    - 최대 25개/공, 오래된 것부터 material.opacity를 0으로 tween 후 scene.remove() + dispose()
    - 잔상 간격으로 낙하 가속을 시각적으로 확인: 아래로 갈수록 간격 넓어짐
  - 바닥 충돌 시:
    - 바운스 효과: y = 0에서 0.15초간 y=0.3까지 올라갔다 내려옴 (sin 기반)
    - 파편 파티클: Points(15개, 0xaaaaaa 회색), 방사형 퍼짐 (초기 속도 Y=2~4, X/Z=-2~2), 0.5초 후 제거
    - 바닥에 먼지 링: RingGeometry(r=0.1→0.5, 0.3초), 0xccccaa, opacity 0.4→0
- 중력 방향 화살표 (토글 ON 시): ArrowHelper, 각 공에서 아래 방향, 0xff0000 빨강, 길이 0.5, "중력 g" 라벨
- 조명: AmbientLight(0.5) + DirectionalLight(0.6, position(5,10,5)) + 그림자 활성화 (공 그림자가 바닥에 보임)
- 카메라: PerspectiveCamera(FOV 50) + OrbitControls, 초기 위치 (8, 12, 8), lookAt(0, 10, 0)

## 낙하 물리 시뮬레이션
- 물리 공식 (교과서 p125~126):
  - 가속도: a = g = 9.8 m/s² (일정)
  - 속도: v(t) = g × t (초기 속도 0)
  - 위치: y(t) = y₀ - ½ × g × t²
- 공기 저항 OFF(기본): 3개 공이 동시에 출발, 동시에 도달
- 공기 저항 ON: 질량이 작은 공이 더 느리게 떨어짐 (항력 = 0.5 × Cd × ρ × A × v²)

## 낙하 애니메이션
1. "낙하 시작" 버튼 클릭
2. 3개 공이 설정된 높이에서 동시에 자유 낙하 시작
3. 각 프레임에서 물리 공식으로 위치 업데이트: y = y₀ - 0.5 × 9.8 × t²
4. 각 공의 현재 속력을 실시간 라벨로 표시: "v = [값] m/s"
5. 바닥(y=0) 도달 시 정지 + 바운스 효과(선택적)
6. 3개 공이 동시에 도달하는 것을 강조: "동시 도달!" 라벨 + 초록 플래시

## 우측 그래프 패널

### 속력-시간 그래프 (교과서 그림 III-15, p125)
- Canvas 2D 렌더링
- X축: 시간 (0~4초), Y축: 속력 (0~39.2 m/s)
- 3개 공의 속력 곡선이 겹침 (공기 저항 OFF 시 완전 동일한 직선)
  - 기울기 = 9.8 m/s² = 중력 가속도
  - 1초: 9.8 m/s, 2초: 19.6 m/s, 3초: 29.4 m/s, 4초: 39.2 m/s
- 각 초마다 점으로 표시 + 속력값 라벨
- 공기 저항 ON 시: 3개 곡선이 분리 (질량 작은 공이 먼저 종단속도에 도달)

### 높이-시간 그래프
- 탭 전환으로 속력-시간 / 높이-시간 그래프 전환
- 높이-시간: 포물선 곡선 (y = h - 0.5gt²)

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "낙하 높이" 슬라이더: 5 ~ 100 (기본값 20), 단위 m, 5 단계
- "낙하 시작" 버튼: 낙하 애니메이션 시작
- "공기 저항" 토글: ON/OFF (기본값 OFF)
  - OFF: 3개 동시 도달 (교과서 핵심)
  - ON: 질량/크기에 따라 도달 시간 차이
- "슬로모션" 슬라이더: 0.1x ~ 2x (기본값 1x)
- "잔상 표시" 토글: ON/OFF (기본값 ON) — 낙하 경로에 반투명 잔상
- "물체 수" 드롭다운: 1개 / 2개 / 3개 (기본값 3개)
- "질량 A" 슬라이더: 0.1 ~ 50 (기본값 1), 단위 kg (물체별 질량 변경)
- "질량 B" 슬라이더: 0.1 ~ 50 (기본값 5)
- "질량 C" 슬라이더: 0.1 ~ 50 (기본값 10)
- "그래프 유형" 드롭다운: 속력-시간 / 높이-시간
- "초기화" 버튼: 공을 초기 높이로 복원

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널로 표시:

```
🍎 자유 낙하 운동
━━━━━━━━━━━━━━
경과 시간:     [t] 초
낙하 높이:     [h] m
현재 속력:     [v] m/s
남은 높이:     [y] m

물리 공식:
  가속도: a = g = 9.8 m/s² (일정)
  속도:   v = g × t = 9.8 × [t] = [v] m/s
  위치:   y = h - ½gt² = [h] - ½×9.8×[t²] = [y] m

핵심: 질량에 관계없이 모든 물체는
      같은 가속도(9.8 m/s²)로 낙하한다!
```
모든 값 실시간 업데이트.

## 교육적 요소
- 핵심 원리: "공기 저항을 무시하면 질량에 관계없이 모든 물체는 같은 가속도(9.8 m/s²)로 자유 낙하합니다. 이것은 갈릴레이가 피사의 사탑 실험으로 보여준 원리입니다."
- 탐구 질문:
  1. "질량이 다른 두 물체를 같은 높이에서 동시에 놓으면 어느 것이 먼저 바닥에 도달하나요?" (답: 동시)
  2. "자유 낙하 하는 물체의 속력이 시간에 따라 일정하게 증가하는 까닭은?" (답: 가속도가 일정하므로)
  3. "실제로 깃털과 볼링공을 떨어뜨리면 볼링공이 먼저 떨어지는 까닭은?" (답: 공기 저항)
  4. "높이 20m에서 자유 낙하 하는 물체가 바닥에 도달하는 시간은?" (답: t=√(2h/g)=√(40/9.8)≈2.02초)
  5. "잔상 간격이 아래쪽으로 갈수록 넓어지는 까닭은?" (답: 속력이 점점 빨라지므로 같은 시간 간격에 더 먼 거리 이동)
- 실생활 예시: 스카이다이빙(공기 저항 → 종단속도), 갈릴레이 피사의 사탑 실험, 아폴로 15호 달에서 깃털-망치 실험, 자유 낙하 놀이기구
- 교과서 연계: "미래엔 통합과학1 III편 p125~126 — 자유 낙하 운동, 중력 가속도(그림 III-15), 탐구: 자유 낙하 하는 물체의 가속도 분석"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원 (OrbitControls 기본 지원), 모바일에서 lil-gui 기본 접힘 상태
- 물리: Clock.getDelta() × 슬로모션 배율, 누적 시간 t += dt, y = y0 - 0.5 * g * t^2, v = g * t
- 잔상: SphereGeometry 재사용(geometry 1개 공유), Material만 clone + opacity 설정
  - 최대 25개/공 × 3공 = 75개 mesh, 오래된 것부터 opacity tween 0 → scene.remove() → material.dispose()
- 그래프: Canvas 2D API (별도 <canvas> 요소, position absolute), 매 프레임 clearRect + 재그리기
- 공기 저항 모드: 항력 F_drag = 0.5 × Cd(0.47) × ρ(1.225) × A(π×r²) × v², 가속도 a = g - F_drag/m
- 바닥 충돌 감지: y <= 0이면 y=0으로 고정, 바운스 또는 정지
- 속력 라벨: CSS2DRenderer 또는 Sprite, 매 프레임 텍스트 업데이트
- 성능: requestAnimationFrame 루프, 총 mesh 최대 ~90개 (공 3 + 잔상 75 + 파편 15)
- 파일명: free_fall_motion_sim.html

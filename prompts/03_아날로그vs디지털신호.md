너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 통합과학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 통합과학1
- 단원: I. 과학의 기초 > 2. 측정 표준과 정보 > 신호와 정보
- 토픽: 아날로그 vs 디지털 신호 변환
- 대상: 고등학교 1학년
- 핵심 개념: 자연계의 신호는 연속적으로 변하는 아날로그 형태이다. 센서를 이용하면 아날로그 형태의 신호를 특정한 값을 갖는 디지털 형태로 측정할 수 있다. 디지털 정보는 저장과 분석이 쉬워 다양한 전자 기기에서 이용된다.

## 씬 구성
- 화면을 수직으로 3단 분할:
  - 상단 (높이 35%): 아날로그 신호 영역
  - 중단 (높이 15%): 센서/변환 과정 영역
  - 하단 (높이 35%): 디지털 신호 영역
  - 최하단 (높이 15%): 실생활 예시 영역
- 상단 아날로그 영역:
  - 2D 캔버스에 연속적인 사인파 렌더링 (Canvas2D 또는 Three.js PlaneGeometry + ShaderMaterial)
  - 파형 색상: 0x3388ff (부드러운 파란색), 선 두께 3px
  - 배경: 0xf8f8ff (연한 라벤더)
  - X축: 시간 (0~24시), Y축: 신호 세기 (0~100%)
  - 축 라벨, 눈금선(GridHelper 스타일 점선) 표시
  - 파형 위 라벨: "아날로그 신호 — 연속적으로 변하는 자연계의 신호" (0x3388ff)
- 중단 센서 영역:
  - 센서 3D 모형: 작은 박스(BoxGeometry 0.5×0.3×0.2) + 렌즈(SphereGeometry 반구) + 전선(TubeGeometry 곡선)
  - 색상: 센서 본체 0x444444 (짙은 회색), 렌즈 0x88aacc (유리색)
  - 센서에서 위(아날로그)→아래(디지털)로 이어지는 화살표 애니메이션 (파티클이 흘러내림)
  - 라벨: "센서 — 자연계의 신호를 전기 신호로 바꾸어 주는 장치"
- 하단 디지털 영역:
  - 아날로그 파형을 샘플링한 결과: 수직 막대(계단 형태)로 표시
  - 막대 색상: 0xff6633 (주황색)
  - 원본 아날로그 파형을 반투명(opacity: 0.2) 파란색으로 겹쳐 표시 → 차이 비교
  - 배경: 0xfff8f0 (연한 크림)
  - 파형 위 라벨: "디지털 신호 — 불연속적인 특정 값으로 변환된 신호" (0xff6633)
- 전체 배경: 0xfafafa (밝은 회색)
- 조명: 불필요 (2D 그래프 중심이므로 AmbientLight(1.0)만)
- 카메라: OrthographicCamera (2D 뷰 기본) + PerspectiveCamera 전환 버튼 (센서 3D 관찰용)

## 샘플링 과정 애니메이션
- "샘플링 시작" 버튼 클릭 시:
  1. 아날로그 파형 위에 수직 스캔 라인(빨간 점선)이 왼쪽에서 오른쪽으로 이동 (속도: 2초/전체)
  2. 스캔 라인이 지나가는 위치마다 파형과의 교점에 빨간 점(CircleGeometry r=5px) 생성
  3. 각 점에서 아래로 수직 점선이 내려가며 하단 디지털 영역에 막대 생성
  4. 샘플링 완료 후 디지털 파형(막대들의 상단을 연결한 계단 형태) 완성
- 샘플링 주파수에 따라 점의 개수 변화:
  - 낮은 주파수(4Hz): 점 4개 → 원본과 매우 다름
  - 중간 주파수(16Hz): 점 16개 → 대략적 형태 유지
  - 높은 주파수(64Hz): 점 64개 → 원본과 거의 동일

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "신호 유형" 드롭다운: 햇빛 세기(사인파) / 기온 변화(사인파+노이즈) / 심박수(톱니파) / 사용자 정의(슬라이더로 주파수/진폭 조절)
- "샘플링 주파수" 슬라이더: 2 ~ 128 (기본값 16), 2의 거듭제곱 단계 (2, 4, 8, 16, 32, 64, 128)
- "비트 수 (양자화)" 슬라이더: 2 ~ 16 (기본값 8), 정수 단계
  - 2비트: 4단계 양자화 → 매우 거친 계단
  - 8비트: 256단계 → 부드러운 계단
  - 16비트: 65536단계 → 거의 연속적
- "샘플링 시작" 버튼: 위에서 설명한 샘플링 애니메이션 재생
- "오버레이 비교" 체크박스: ON 시 하단에 원본 아날로그 파형을 반투명으로 겹침
- "초기화" 버튼: 기본값으로 리셋

## 시각화 규칙

### 샘플링 주파수에 따른 시각 피드백
- 주파수 ≤ 8: 디지털 파형 주위 빨간 테두리 + 경고 텍스트 "⚠️ 샘플링 부족 — 원본 신호 손실 큼"
- 주파수 9~32: 노란 테두리 + "⚡ 적절한 샘플링"
- 주파수 ≥ 33: 초록 테두리 + "✅ 고품질 디지털 변환"

### 비트 수에 따른 시각 피드백
- 비트 수 ≤ 4: 계단 높이 크게 보임, 양자화 노이즈 표시 (원본과의 차이를 빨간 영역으로 채움)
- 비트 수 ≥ 8: 양자화 노이즈 거의 안 보임

### 오차율 표시
- 하단 좌측에 "디지털 변환 오차율: XX.X%" 실시간 계산 표시
- 오차율 = 원본과 디지털의 RMS 차이 / 원본 RMS × 100
- 오차율에 따라 색상 변화: >20% 빨강, 5~20% 노랑, <5% 초록

## 실생활 예시 영역 (최하단)
교과서 그림 I-7 (p33) 기반:
- 좌측: 아날로그 속력계 이미지(원형 다이얼, 바늘) + "아날로그" 라벨
- 우측: 디지털 속력계 이미지(LED 숫자 표시) + "디지털" 라벨
- 속력값 슬라이더 (0~200 km/h): 두 속력계에 동시 반영
  - 아날로그: 바늘이 부드럽게 회전
  - 디지털: 숫자가 정수로 변경 (불연속적)
- 차이점 강조 텍스트: "아날로그는 연속적 변화, 디지털은 불연속적 값"

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널로 표시:

```
📡 아날로그 → 디지털 변환
━━━━━━━━━━━━━━━━━━━
샘플링 주파수:  [값] Hz (초당 [값]번 측정)
비트 수:       [값] bit (2^[값] = [값]단계 양자화)
샘플 수:       [값]개
양자화 단계:   [값]단계
변환 오차율:   [값]%

나이퀴스트 정리:
f_sampling ≥ 2 × f_signal (원본 주파수의 2배 이상 샘플링해야 원본 복원 가능)
```
모든 값은 슬라이더 조작 시 실시간 업데이트.

## 교육적 요소
화면 상단에 토글 가능한 "📖 학습 가이드" 패널:
- 핵심 원리: "자연계의 신호는 연속적으로 변하는 아날로그 형태입니다. 센서를 이용하면 이를 디지털 형태로 변환할 수 있으며, 디지털 정보는 저장·분석·전송이 쉬워 현대 문명에 큰 영향을 미쳤습니다."
- 탐구 질문:
  1. "샘플링 주파수를 낮추면 디지털 신호는 원본과 어떻게 달라지나요?"
  2. "비트 수를 높이면 왜 원본에 더 가까운 디지털 신호를 얻을 수 있나요?"
  3. "CD 음악은 44,100Hz로 샘플링합니다. 사람이 들을 수 있는 최대 주파수(20,000Hz)와 어떤 관계가 있을까요?"
- 실생활 예시: CD 음악(44.1kHz, 16bit), 전화 음성(8kHz, 8bit), 4K 영상(디지털 이미지 센서), 스마트워치 심박 센서
- 교과서 연계: "미래엔 통합과학1 I편 p32~34 — 신호와 정보, 센서, 디지털 정보와 현대 문명"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- 2D 그래프 부분은 HTML Canvas 2D API 사용 가능 (Three.js와 혼합)
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원
- 성능: requestAnimationFrame 루프, 샘플링 애니메이션은 별도 타이머
- 파일명: analog_digital_signal_sim.html

너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 통합과학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 통합과학1
- 단원: I. 과학의 기초 > 2. 측정 표준과 정보 > 신호와 정보
- 토픽: 센서와 신호 측정
- 대상: 고등학교 1학년
- 핵심 개념: 센서는 자연계의 신호를 받아들여 전기 신호로 바꾸어 주는 장치이다. 신호를 측정해 분석하면 유용한 정보를 얻을 수 있다. 체온을 측정해 건강 상태를 알 수 있고, 우주에서 온 빛을 연구해 우주의 생성 과정을 알 수 있다.

## 씬 구성
- 화면 중앙에 3D 환경 장면을 렌더링하고, 그 안에서 센서가 작동하는 모습을 시각화한다.
- 3가지 시나리오를 탭으로 전환:

### 시나리오 A: 하루의 빛 변화 + 빛 센서
- 3D 장면: 지평선이 보이는 야외 풍경
  - 바닥: PlaneGeometry(50, 50), MeshStandardMaterial(0x4a7c3f 풀밭 초록)
  - 하늘: 반구 SphereGeometry(25, 32, 32, 0, Math.PI*2, 0, Math.PI/2), 내부면 렌더링
  - 하늘색은 시간에 따라 변화: 밤(0x0a0a2e) → 새벽(0xff7744) → 낮(0x87CEEB) → 저녁(0xff5533) → 밤
  - 태양: SphereGeometry(1, 32, 32), emissive 0xffdd00, 시간에 따라 호를 그리며 이동
  - 태양 위치: 0시=지평선 아래 / 6시=동쪽 지평선 / 12시=천정 / 18시=서쪽 지평선 / 24시=아래
- 빛 센서 3D 모형: 작은 박스(BoxGeometry 0.3×0.2×0.1) + 볼록 렌즈(SphereGeometry 반구 r=0.08) + 전선(TubeGeometry)
  - 센서 위치: 바닥 위 기둥(CylinderGeometry) 꼭대기, 센서 렌즈가 하늘을 향함
  - 색상: 본체 0x333333, 렌즈 0x88aadd
- 센서에서 우측으로 연결된 실시간 그래프 패널 (HTML overlay):
  - X축: 시간 (0~24시), Y축: 빛 세기 (0~100%, lux)
  - 시간 경과에 따라 그래프가 실시간으로 그려짐
  - 밤: ~0%, 새벽: 급상승, 정오: ~100%, 저녁: 급하강
- 조명: AmbientLight(시간에 따라 0.05~0.6 변화) + DirectionalLight(태양, 시간에 따라 위치/세기 변화)

### 시나리오 B: 계절 기온 변화 + 온도 센서
- 3D 장면: 나무 한 그루가 있는 풍경
  - 바닥: PlaneGeometry, 계절에 따라 색상 변화 (봄: 연초록 → 여름: 진초록 → 가을: 주황갈색 → 겨울: 흰색)
  - 나무: CylinderGeometry(줄기) + SphereGeometry(잎), 계절에 따라 잎 색상/크기 변화
    - 봄: 연분홍(꽃) 0xffb7c5 / 여름: 진초록 0x228b22 / 가을: 주황 0xd2691e / 겨울: 잎 없음(구 제거) + 눈 파티클
  - 눈 파티클: Points(200개), 겨울에만 위에서 아래로 내림
- 온도 센서 3D 모형: 수은 온도계 스타일 — CylinderGeometry(가느다란 관) + SphereGeometry(구부)
  - 빨간 수은(관 내부): 온도에 따라 높이 변화 (MeshBasicMaterial 0xff0000)
  - 눈금: -20°C ~ 40°C, 5°C 간격 라벨
- 실시간 그래프:
  - X축: 월 (1~12월), Y축: 기온 (-10°C ~ 35°C)
  - 서울 평균 기온 기반: 1월 -2°C, 4월 12°C, 7월 25°C, 10월 15°C (사인파 근사)

### 시나리오 C: 열화상 카메라 (교과서 그림 I-8 기반)
- 3D 장면: 3명의 사람 실루엣(BoxGeometry 조합으로 단순화된 인체)이 나란히 서 있음
  - 사람 1: 정상 체온 36.5°C
  - 사람 2: 미열 37.5°C
  - 사람 3: 발열 38.5°C
- 열화상 뷰 토글:
  - 일반 뷰: 사람 실루엣이 평범한 색상(0xddccbb 피부색)
  - 열화상 뷰: 체온에 따라 컬러맵 적용
    - 35°C 이하: 파랑 (0x0000ff)
    - 36~37°C: 초록→노랑 (0x00ff00 → 0xffff00)
    - 37~38°C: 주황 (0xff8800)
    - 38°C 이상: 빨강 (0xff0000)
  - 각 사람 위에 온도 라벨: "36.5°C ✅", "37.5°C ⚠️", "38.5°C 🔴"
- 교과서: "열화상 카메라로 피부 온도를 측정, 분석해 체온이 높은 사람을 확인할 수 있다" (p33 그림 I-8)

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "시나리오 선택" 드롭다운: 빛 센서 (하루) / 온도 센서 (계절) / 열화상 카메라
- 시나리오 A 전용:
  - "시각" 슬라이더: 0 ~ 24 (기본값 12), 0.5 단계, 단위 "시"
  - "자동 재생" 버튼: 0시→24시 자동 진행 (10초 동안)
- 시나리오 B 전용:
  - "월" 슬라이더: 1 ~ 12 (기본값 7), 정수 단계
  - "자동 재생" 버튼: 1월→12월 자동 진행 (12초 동안)
- 시나리오 C 전용:
  - "열화상 뷰" 토글: ON/OFF (기본값 OFF)
  - "사람 1 체온" 슬라이더: 35.0 ~ 40.0 (기본값 36.5), 0.1 단계
  - "사람 2 체온" 슬라이더: 35.0 ~ 40.0 (기본값 37.5), 0.1 단계
  - "사람 3 체온" 슬라이더: 35.0 ~ 40.0 (기본값 38.5), 0.1 단계
- "초기화" 버튼: 각 시나리오 기본값 리셋

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널로 표시:

```
📡 센서와 신호 측정
━━━━━━━━━━━━━━━
센서 종류:   [빛 센서 / 온도 센서 / 열화상 카메라]
입력 신호:   [빛 세기 / 기온 / 적외선]
출력 신호:   전기 신호 (전압)
현재 측정값: [값] [단위]

🔄 신호 → 정보 흐름:
자연계 변화 → 센서(전기 신호 변환) → 측정·분석 → 유용한 정보
```
시나리오 및 슬라이더 변경 시 실시간 업데이트.

## 교육적 요소
화면 상단에 토글 가능한 "📖 학습 가이드" 패널:
- 핵심 원리: "센서는 자연계의 신호를 받아들여 전기 신호로 바꾸어 주는 장치입니다. 신호를 측정해 분석하면 건강 상태, 날씨, 우주의 비밀 등 유용한 정보를 얻을 수 있습니다."
- 탐구 질문:
  1. "빛 센서가 없다면 하루 동안의 빛 변화를 어떻게 기록할 수 있을까요?"
  2. "열화상 카메라는 어떤 종류의 센서를 사용할까요? (힌트: 적외선)"
  3. "스마트워치는 어떤 센서로 심박수를 측정할까요?"
- 센서 종류 추가 정보 (교과서 p33): 빛 센서, 온도 센서, 습도 센서, 압력 센서, 가속도 센서
- 실생활 예시 (교과서 그림 I-8):
  - 우주 망원경: 우주에서 온 빛 → 우주 생성 과정 정보
  - 열화상 카메라: 피부 온도 → 발열자 확인
  - 지진계: 지진파 → 지진 발생 위치 정보
- 교과서 연계: "미래엔 통합과학1 I편 p32~34 — 신호의 측정과 분석, 센서, 디지털 정보와 현대 문명"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원 (OrbitControls 기본 지원)
- 시나리오 전환 시 이전 씬 객체 dispose() + 새 씬 구성
- 성능: requestAnimationFrame 루프, 파티클(눈) 최대 200개
- 파일명: sensor_signal_measurement_sim.html

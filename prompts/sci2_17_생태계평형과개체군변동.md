너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 통합과학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 통합과학2
- 단원: II. 환경과 에너지 > 1. 생태계와 환경 > 생태계평형
- 토픽: 생태계 평형과 개체군 변동
- 대상: 고등학교 1학년
- 핵심 개념: 생태계를 구성하는 생물의 종류, 개체수, 에너지의 이동이 안정적으로 유지되는 상태를 생태계평형이라고 한다. 일반적으로 생태계는 스스로 평형을 유지할 수 있지만 회복 능력을 넘어서는 수준으로 평형이 깨지면 쉽게 회복하기 어렵다. 미국 카이바브 고원에서 늑대를 사냥하자 사슴 개체수가 급증했다가 먹이 부족으로 급감한 사례가 있다.

## 씬 구성
- 화면 좌측 60%: 3D 초원 생태계 (풀, 사슴, 늑대)
- 화면 우측 40%: 개체군 변동 그래프 (Canvas 2D, 교과서 p78 카이바브 고원 데이터)
- 배경: 0x88bb77 (초원 녹색 하늘)
- 카메라: PerspectiveCamera(FOV 55) + OrbitControls (위에서 비스듬히)
- 조명: AmbientLight(0.4) + DirectionalLight(0.7)

## 카이바브 고원 시뮬레이션 (교과서 p78)

### 3D 초원
- 바닥: PlaneGeometry(30, 30), 0x66aa44 (초원)
- 풀: ConeGeometry 작은 것 100~200개 랜덤 배치, 0x55aa33
- 나무: CylinderGeometry + SphereGeometry, 5~8그루, 0x338822
- 산(배경): ConeGeometry 큰 것 2~3개, 0x887766, 먼 배경에 배치

### 사슴 (Deer)
- BoxGeometry 조합: 몸통(0xcc9966) + 머리 + 다리 4개 + 뿔(CylinderGeometry 분기)
- 초기 개체수: ~10마리 (3D 공간에 분산)
- 풀을 먹는 애니메이션 (머리를 아래로 숙이는 모션)
- 이동: 랜덤 방향으로 천천히 이동

### 늑대 (Wolf)
- BoxGeometry 조합: 몸통(0x777788) + 머리 + 다리 4개 + 꼬리
- 초기 개체수: ~3마리
- 사슴을 추적하는 이동 패턴 (가장 가까운 사슴 방향으로)
- 사슴에 접근하면 포식 이벤트 (사슴 제거 + 작은 파티클 효과)

### 풀(식물군집)
- 풀의 양을 시각적으로 표현 (풀 개수 또는 높이 변화)
- 사슴이 많으면 풀 감소, 사슴이 줄면 풀 회복

## 시뮬레이션 메커니즘

### 시나리오 1: 정상 생태계 (1905년 이전)
- 늑대 ~400마리, 사슴 ~3800마리 (스케일 다운: 늑대 3, 사슴 10)
- 안정적인 평형 상태 유지
- 늑대가 사슴을 포식 → 사슴 개체수 적정 유지 → 풀도 적정 유지

### 시나리오 2: 늑대 사냥 허용 (1905~1920년)
- 늑대 개체수 감소 (3 → 0)
- 사슴 개체수 급증 (10 → 30~40)
- 풀/식물군집 양 감소 (사슴의 과잉 방목)

### 시나리오 3: 사슴 개체수 붕괴 (1920~1930년)
- 먹이(풀) 부족으로 사슴 개체수 급감 (40 → 5)
- 식물군집이 서서히 회복

### 시나리오 전환
- 자동 재생: 1905년부터 1930년까지 시간 경과에 따라 개체수 변동
- 또는 연도 슬라이더로 직접 조절

## 개체군 변동 그래프 (교과서 p78)
- Canvas 2D:
  - X축: 연도 (1905~1930)
  - Y축 (왼쪽): 개체수 (마리)
  - 3개 곡선:
    - 사슴: 빨간 실선 (1905년 ~3800 → 1920년 ~100,000 → 1930년 급감)
    - 늑대: 파란 점선 (1905년 ~400 → 1920년 거의 0)
    - 식물군집: 초록 막대 (사슴과 반대 경향)
  - 현재 시점을 세로 점선으로 표시
- 교과서: "1905년 미국의 카이바브 고원에는 약 400마리의 늑대와 약 3800마리의 사슴이 살고 있었다. 미국 정부는 사슴을 보호하기 위해 사슴의 포식자인 늑대 사냥을 허용했다" (p78)

## 생태계평형 회복 과정 (교과서 그림 II-7, p77)
- 3단계 순환 다이어그램:
  1. 1차 소비자 증가 → 생산자 감소, 2차 소비자 증가
  2. 생산자 감소 + 2차 소비자 증가 → 1차 소비자 감소
  3. 1차 소비자 감소 → 생산자 회복, 2차 소비자 감소 → 원래 상태

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "연도" 슬라이더: 1905~1930 (기본값 1905)
- "자동 재생" 버튼: 1905→1930 자동 진행 (25초)
- "재생 속도" 슬라이더: 1x~5x
- "늑대 사냥" 토글: ON/OFF (1905년에 ON으로 시작 시 늑대 사냥 시작)
- "개체수 표시" 토글: ON/OFF (각 동물 위에 숫자)
- "초기화" 버튼: 1905년 상태로 리셋

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원 (OrbitControls 기본 지원)
- InstancedMesh로 사슴/늑대/풀 효율적 렌더링
- 연도 변경 시 개체수 갱신 (dispose 없이 속성 변경)
- gui.controllersRecursive().forEach(c => c.updateDisplay()) 동기화
- 성능: requestAnimationFrame 루프
- 파일명: sci2_17_ecosystem_equilibrium.html

너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 통합과학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 통합과학2
- 단원: II. 환경과 에너지 > 1. 생태계와 환경 > 생태계평형
- 토픽: 먹이 관계와 에너지 피라미드
- 대상: 고등학교 1학년
- 핵심 개념: 생태계에서 먹이 관계는 그물처럼 복잡하게 형성되어 있는데, 이를 먹이 그물이라고 한다. 에너지양과 개체수가 상위 영양단계로 갈수록 점점 줄어드는 형태를 그림으로 나타낸 것을 생태피라미드라고 한다. 안정된 생태계는 상위 영양단계로 갈수록 에너지양과 개체수가 줄어드는 피라미드 형태를 유지한다.

## 씬 구성
- 화면 좌측 60%: 3D 먹이 그물 시각화 (생물 모형 + 화살표 연결)
- 화면 우측 40%: 에너지 피라미드 (HTML overlay 또는 3D)
- 배경: 0x223322 (어두운 숲 배경)
- 카메라: PerspectiveCamera(FOV 55) + OrbitControls
- 조명: AmbientLight(0.3) + DirectionalLight(0.6)

## 먹이 그물 3D

### 생산자 (하단)
- 풀: ConeGeometry 여러 개, 0x44aa33
- 나무: CylinderGeometry + SphereGeometry, 0x338822
- 해초: CylinderGeometry 물결 형태, 0x55bb55

### 1차 소비자 (중하단)
- 메뚜기: BoxGeometry 작은 곤충, 0x55aa33
- 토끼: SphereGeometry 몸 + 귀, 0xddccaa
- 쥐: SphereGeometry 작은 몸, 0x888877

### 2차 소비자 (중상단)
- 개구리: SphereGeometry, 0x44aa44
- 뱀: CylinderGeometry 긴 형태, 0x667744
- 참새: BoxGeometry 조합, 0x887766

### 3차 소비자 (상단)
- 매: BoxGeometry 조합 날개 펼친 형태, 0x554433, 크기 큰
- 여우: BoxGeometry 조합, 0xcc7733

### 분해자
- 버섯: CylinderGeometry + SphereGeometry, 0xaa7744, 하단 가장자리

### 먹이 관계 화살표
- 각 먹이 관계를 화살표(TubeGeometry 곡선 + ConeGeometry 화살촉)로 연결
- 화살표 방향: 먹히는 생물 → 먹는 생물 (에너지 흐름 방향)
- 화살표 색상: 영양단계별 구분 (초록→노랑→주황→빨강)
- 특정 생물 클릭 시 해당 생물과 연결된 먹이 관계만 강조, 나머지 반투명
- 먹이 사슬: 단순 직선 연결 (1차원)
- 먹이 그물: 복잡하게 교차하는 연결 (그물 형태)

## 에너지 피라미드 (우측)

### 피라미드 구조
- 4층 피라미드 (BoxGeometry, 아래로 갈수록 넓음):
  - 3차 소비자: 상단, 폭 1, 높이 0.5, 0xff4444 (빨강)
  - 2차 소비자: 폭 2, 높이 0.6, 0xff8844 (주황)
  - 1차 소비자: 폭 3, 높이 0.7, 0xffcc44 (노랑)
  - 생산자: 하단, 폭 4, 높이 0.8, 0x44aa44 (초록)

### 피라미드 정보
- 각 층에 라벨: 영양단계 이름 + 대표 생물 + 에너지양(상대값)
- 에너지양: 생산자 100% → 1차 소비자 ~10% → 2차 소비자 ~1% → 3차 소비자 ~0.1%
- "상위 영양단계로 갈수록 에너지양과 개체수가 줄어든다" 강조 텍스트
- 피라미드 전환: 에너지 피라미드 / 개체수 피라미드 / 생물량 피라미드

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "표시 모드" 드롭다운: 먹이 사슬 / 먹이 그물 / 에너지 피라미드 / 전체
- "피라미드 종류" 드롭다운: 에너지 / 개체수 / 생물량
- "특정 생물 강조" 드롭다운: 전체 / 메뚜기 / 토끼 / 개구리 / 매 / ...
- "에너지 흐름 애니메이션" 버튼: 생산자→3차 소비자까지 에너지 이동 파티클 표시
- "초기화" 버튼

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널로 표시:

```
🔗 먹이 관계와 에너지 피라미드
━━━━━━━━━━━━━━━━━━━
현재 보기:      [먹이 사슬/그물/피라미드]

영양단계별 에너지양:
  생산자:      100% (10,000 kcal)
  1차 소비자:  ~10% (1,000 kcal)
  2차 소비자:  ~1%  (100 kcal)
  3차 소비자:  ~0.1% (10 kcal)

규칙:
  상위 영양단계로 갈수록
  에너지양과 개체수가 줄어든다
  → 생태피라미드 형태
```

## 교육적 요소
화면 상단에 토글 가능한 "📖 학습 가이드" 패널:
- 핵심 원리: "생태계에서 먹이 관계는 그물처럼 복잡하게 형성되어 먹이 그물이라 합니다. 에너지양과 개체수가 상위 영양단계로 갈수록 줄어드는 형태를 생태피라미드라고 합니다."
- 탐구 질문:
  1. "생산자의 에너지가 1차 소비자로 전달될 때 약 10%만 전달되는 까닭은 무엇인가요?"
  2. "먹이 사슬과 먹이 그물의 차이점은 무엇인가요?"
  3. "3차 소비자(매)의 개체수가 갑자기 감소하면 생태계에 어떤 변화가 일어날까요?"
- 실생활 예시: 식량 생산 효율(곡물 직접 소비 vs 가축 사육), 멸종위기종 보호
- 교과서 연계: "미래엔 통합과학2 II편 p76~77 — 먹이 관계, 에너지 피라미드, 생태피라미드"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원 (OrbitControls 기본 지원)
- 모드 전환 시 이전 객체 dispose() + 화살표/강조 상태 리셋
- gui.controllersRecursive().forEach(c => c.updateDisplay()) 동기화
- 성능: requestAnimationFrame 루프
- 파일명: sci2_16_food_web_energy_pyramid.html

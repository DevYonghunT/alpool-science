너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 물리학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 전자기와 양자
- 단원: III. 양자와 미시세계 > 04. 현대의 원자 모형
- 토픽: 전자 배치와 주기율표
- 대상: 고등학교 (진로 선택과목)
- 핵심 개념: 다전자 원자에서 전자는 에너지가 낮은 오비탈부터 채워진다(쌓음 원리, Aufbau). 파울리 배타 원리에 의해 하나의 오비탈에는 스핀이 반대인 전자 최대 2개까지만 들어갈 수 있다. 훈트 규칙에 따라 같은 에너지의 오비탈에는 전자가 하나씩 먼저 채워진 후 쌍을 이룬다. 이 규칙들이 원소의 전자 배치를 결정하고, 주기율표의 주기성을 설명한다.

## 씬 구성
- 화면 좌측 50%: 3D 에너지 준위 + 전자 배치 시각화
- 화면 우측 50%: 주기율표 + 전자 배치 표기 (HTML overlay)
- 배경: 0x0a0a1a
- 카메라: PerspectiveCamera(FOV 50) + OrbitControls
- 조명: AmbientLight(0.4) + DirectionalLight(0.4)

## 3D 에너지 준위와 전자 배치

### 에너지 준위 (3D 계단 구조)
- 수직축: 에너지 (y축, 위로 갈수록 높음)
- 각 오비탈: BoxGeometry(가로 1, 세로 0.1, 깊이 0.8), 오비탈 유형별 색상
  - s 오비탈: 0x4488ff (파랑)
  - p 오비탈: 0x44cc88 (초록), 3개 나란히
  - d 오비탈: 0xffaa44 (주황), 5개 나란히
  - f 오비탈: 0xff6688 (분홍), 7개 나란히
- 채움 순서: 1s → 2s → 2p → 3s → 3p → 4s → 3d → 4p → ...
  - 화살표로 채움 순서 표시 (ArrowHelper, 0xffdd44)

### 전자
- SphereGeometry(r=0.12), 0xffff44 emissive 0xffdd00
- 스핀 ↑: 위쪽 화살표 표시 (작은 ArrowHelper)
- 스핀 ↓: 아래쪽 화살표 표시
- 오비탈 박스 위에 전자가 하나씩 배치되는 애니메이션

### 파울리 배타 원리 시각화
- 하나의 오비탈 박스에 전자 2개까지 (↑↓)
- 3번째 전자 추가 시도 → 빨간 "✕" + 경고 + 바운스 애니메이션
  - "파울리 배타 원리: 같은 양자수 조합 불가!" 라벨
- 같은 스핀 2개 시도 → 빨간 "✕"
  - "같은 스핀은 같은 오비탈에 들어갈 수 없다!" 라벨

### 훈트 규칙 시각화
- p 오비탈 3개 (같은 에너지): px, py, pz
- 전자 채움 순서 (예: 질소 N, 7전자):
  - 1s↑↓ → 2s↑↓ → 2p: px↑, py↑, pz↑ (하나씩 먼저!)
  - 산소 O(8전자): px↑↓, py↑, pz↑ (이제야 쌍)
- "훈트 규칙: 같은 에너지 오비탈에 홀전자 먼저!" 라벨
- 잘못된 배치 시도(한 곳에 2개 먼저) → 경고 + 올바른 배치로 수정 애니메이션

### 쌓음 원리(Aufbau) 애니메이션
- "채우기" 버튼: 선택한 원소의 전자를 하나씩 낮은 에너지부터 채움
- 전자가 날아와서 오비탈에 안착하는 애니메이션
- 각 단계에서 적용된 원리 표시

## 주기율표 (우측 HTML overlay)

### 미니 주기율표
- 1~36번 원소 (수소~크립톤) 격자 표시
- 각 칸: 원소 기호 + 원자 번호
- 현재 선택 원소 하이라이트 (노란 테두리)
- 전자 배치: 1s² 2s² 2p⁶ ... 형태로 표시
- 블록 구분 색상:
  - s-블록: 0x4488ff (1,2족)
  - p-블록: 0x44cc88 (13~18족)
  - d-블록: 0xffaa44 (3~12족)
- 클릭하면 해당 원소의 3D 전자 배치 표시

### 전자 배치 표기
- 궤도 표기법: 1s² 2s² 2p⁶ 3s² ...
- 화살표 표기법: ↑↓ ↑↓ ↑↓↑↓↑↓ ...
- 약식 표기: [Ne] 3s² 3p³ (인의 경우)

## 주기성 시각화
- 원자 번호 vs 이온화 에너지 그래프 (Canvas 2D)
  - s, p 껍질 완성 시 높은 이온화 에너지 (비활성 기체)
  - 주기적 패턴 확인
- 최외각 전자 수 → 화학적 성질 연결
  - 1족: 1개 → 반응성 큼
  - 18족: 8개(채움) → 안정

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "원소" 드롭다운: H(1) ~ Kr(36) (기본값 C(6))
- "원자 번호 Z" 슬라이더: 1~36 (드롭다운과 연동)
- "채우기 애니메이션" 버튼: 전자를 하나씩 채움
- "속도" 슬라이더: 0.5x~3x (애니메이션 속도)
- "파울리 위반 시도" 버튼: 위반 상황 데모
- "훈트 규칙 데모" 버튼: 올바른 vs 잘못된 배치 비교
- "이온화 에너지 그래프" 토글: ON/OFF
- "주기율표" 토글: ON/OFF (기본값 ON)
- "초기화" 버튼

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널로 표시:

```
🧱 전자 배치와 주기율표
━━━━━━━━━━━━━━━━━
원소: [기호] (Z=[값])

전자 배치:
  [1s² 2s² 2p⁶ ...]
  약식: [[비활성기체] + 나머지]

세 가지 규칙:
  ① 쌓음 원리: 낮은 에너지부터 채움
     1s→2s→2p→3s→3p→4s→3d→4p
  ② 파울리 배타 원리: 
     1 오비탈 → 최대 2전자 (↑↓)
  ③ 훈트 규칙:
     같은 에너지 → 홀전자 먼저

최외각 전자: [N]개 → [족] 성질
```

## 교육적 요소
화면 상단에 토글 가능한 "📖 학습 가이드" 패널:
- 핵심 원리: "전자 배치는 쌓음 원리, 파울리 배타 원리, 훈트 규칙의 세 가지 규칙으로 결정됩니다. 이 규칙들이 주기율표의 주기성과 원소의 화학적 성질을 설명합니다."
- 탐구 질문:
  1. "4s 오비탈이 3d 오비탈보다 먼저 채워지는 까닭은 무엇인가요?"
  2. "질소(N)와 산소(O)의 전자 배치에서 훈트 규칙이 어떻게 적용되나요?"
  3. "비활성 기체(He, Ne, Ar)의 이온화 에너지가 높은 까닭을 전자 배치로 설명해 보세요."
- 실생활 예시: 네온사인(전자 전이), 반도체(전자 배치와 밴드 구조), 화학 반응(최외각 전자), 자성(홀전자)
- 교과서 연계: "전자기와 양자 III편 p112~117 — 전자 배치, 파울리 배타 원리, 훈트 규칙"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원 (OrbitControls 기본 지원)
- 원소 변경 시 이전 객체 dispose() + 새 객체 생성
- gui.controllersRecursive().forEach(c => c.updateDisplay()) 동기화
- 성능: requestAnimationFrame 루프
- 파일명: emq_26_electron_configuration.html

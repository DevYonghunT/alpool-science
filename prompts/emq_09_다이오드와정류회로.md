너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 물리학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 전자기와 양자
- 단원: I. 전자기적 상호작용 > 06. 반도체 소자를 활용한 전자 회로
- 토픽: 다이오드와 정류 회로
- 대상: 고등학교 (진로 선택과목)
- 핵심 개념: 다이오드는 p-n 접합 반도체로 순방향에서만 전류를 통과시키는 정류 작용을 한다. 반파 정류는 다이오드 1개로 교류의 반쪽만 통과시키고, 전파 정류는 다이오드 4개(브릿지 회로)로 교류의 양쪽을 모두 같은 방향의 전류로 변환한다. 축전기를 추가하면 맥동이 줄어든 직류를 얻을 수 있다.

## 씬 구성
- 화면 좌측 50%: 3D 정류 회로 (다이오드 + 교류 전원 + 부하)
- 화면 우측 50%: 입력(교류) vs 출력(정류) 파형 그래프 (Canvas 2D)
- 3가지 모드: 반파 정류 / 전파 정류(브릿지) / 전파 정류 + 평활
- 배경: 0xeef0f5
- 카메라: PerspectiveCamera(FOV 50) + OrbitControls
- 조명: AmbientLight(0.5) + DirectionalLight(0.6)

## 3D 정류 회로

### 회로 소자
- 교류 전원: CircleGeometry + "~" 표시, 0xdddddd
- 다이오드: BoxGeometry(0.3, 0.15, 0.15) + 삼각형(화살표 방향), 0x333333
  - 순방향: 전류 통과 (초록 표시)
  - 역방향: 전류 차단 (빨간 X)
- 저항(부하): CylinderGeometry 색띠, 0xaa8844
- 축전기(평활용): 2개 평행판, 0x4488aa
- 전선: TubeGeometry로 연결

### 전류 흐름 시각화
- 전류: 노란 파티클이 전선을 따라 이동
- 교류: 왕복 이동 (방향이 주기적으로 바뀜)
- 다이오드 통과: 순방향 반주기에만 파티클 통과
- 역방향 반주기: 파티클 차단 (다이오드에서 멈춤)

## 모드 1: 반파 정류

### 회로
- 교류 전원 + 다이오드 1개 + 저항
- 양(+) 반주기: 다이오드 순방향 → 전류 통과 → 저항에 전압
- 음(−) 반주기: 다이오드 역방향 → 전류 차단 → 저항에 전압 0

### 파형 (우측 그래프)
- 입력: 사인파 (양/음 모두)
- 출력: 양(+) 반주기만 남은 반파 (음(−)은 0)
- "교류의 반쪽만 통과" 라벨

## 모드 2: 전파 정류 (브릿지 회로)

### 회로
- 교류 전원 + 다이오드 4개(브릿지 배치) + 저항
- 양(+) 반주기: D₁, D₃ 통과 → 저항에 양(+) 전류
- 음(−) 반주기: D₂, D₄ 통과 → 저항에 여전히 양(+) 전류!
- 3D에서 각 반주기마다 어떤 다이오드가 활성인지 색상으로 표시
  - 활성 다이오드: 초록 발광
  - 비활성 다이오드: 어두운 회색

### 파형
- 입력: 사인파
- 출력: 음(−) 반주기가 위로 뒤집힌 전파 정류파 (맥동 직류)
- "교류의 양쪽 모두 같은 방향으로 변환" 라벨

## 모드 3: 전파 정류 + 평활 (축전기 추가)

### 회로
- 전파 정류 + 축전기 병렬 연결
- 축전기가 충전/방전 반복 → 맥동 감소 → 거의 직류에 가까움

### 파형
- 전파 정류파 위에 축전기 효과:
  - 전압 상승 시: 축전기 충전 (전압 따라감)
  - 전압 하강 시: 축전기 방전 (전압 느리게 감소)
  - → 맥동이 줄어든 평탄한 직류
- C값 슬라이더: C↑ → 맥동↓ → 더 평탄한 직류
- "축전기가 클수록 맥동이 줄어 안정적인 직류" 라벨

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "모드" 드롭다운: 반파 정류 / 전파 정류 / 전파 + 평활
- "교류 전압" 슬라이더: 1~12V (기본값 6)
- "교류 진동수" 슬라이더: 1~10Hz (시각화용, 기본값 2)
- "축전기 C" 슬라이더 (모드 3): 1~100μF (기본값 10)
- "전류 흐름 표시" 토글: ON/OFF (기본값 ON)
- "다이오드 활성 표시" 토글: ON/OFF (기본값 ON)
- "시뮬레이션 시작" 버튼
- "초기화" 버튼

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널로 표시:

```
🔌 다이오드와 정류 회로
━━━━━━━━━━━━━━━
모드: [반파/전파/전파+평활]
V_AC = [값]V, f = [값]Hz

다이오드 정류 작용:
  순방향 → 전류 통과 ✅
  역방향 → 전류 차단 ❌

반파 정류:
  다이오드 1개, 양(+) 반주기만 통과
  출력: 반파 맥동 직류

전파 정류 (브릿지):
  다이오드 4개, 양쪽 모두 같은 방향
  출력: 전파 맥동 직류

평활 (축전기):
  C = [값]μF
  C↑ → 맥동↓ → 안정 직류
```

## 교육적 요소
화면 상단에 토글 가능한 "📖 학습 가이드" 패널:
- 핵심 원리: "다이오드는 순방향에서만 전류를 통과시킵니다. 반파 정류는 교류의 반쪽만, 전파 정류(브릿지)는 양쪽 모두를 같은 방향 전류로 변환합니다. 축전기를 추가하면 맥동이 줄어든 직류를 얻습니다."
- 탐구 질문:
  1. "반파 정류와 전파 정류의 출력 파형 차이를 설명해 보세요."
  2. "브릿지 정류 회로에서 양(+) 반주기와 음(−) 반주기에 각각 어떤 다이오드가 활성인가요?"
  3. "평활 축전기의 용량을 키우면 출력이 어떻게 변하나요?"
- 실생활 예시: 스마트폰 충전기(AC→DC), 컴퓨터 파워 서플라이, LED 전구 드라이버
- 교과서 연계: "전자기와 양자 I편 p38~43 — 반도체 소자를 활용한 전자 회로, 다이오드"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원 (OrbitControls 기본 지원)
- 모드 전환 시 이전 객체 dispose() + 새 객체 생성
- gui.controllersRecursive().forEach(c => c.updateDisplay()) 동기화
- 성능: requestAnimationFrame 루프, 전류 파티클 최대 30개
- 파일명: emq_09_diode_rectifier.html

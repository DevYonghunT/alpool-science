너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 물리학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 전자기와 양자
- 단원: I. 전자기적 상호작용 > 06. 반도체 소자를 활용한 전자 회로 / 프로젝트
- 토픽: IoT와 마이크로프로세서
- 대상: 고등학교 (진로 선택과목)
- 핵심 개념: 사물 인터넷(IoT)은 다양한 기기가 인터넷에 연결되어 데이터를 주고받는 기술이다. 마이크로프로세서(아두이노 등)는 센서 입력을 받아 처리하고 출력 장치를 제어하는 소형 컴퓨터이다. 센서(입력) → 마이크로프로세서(처리) → 액추에이터(출력)의 구조로 다양한 자동 제어 시스템을 구현한다.

## 씬 구성
- 화면 중앙: 3D 스마트 홈 환경 (센서 + 마이크로프로세서 + 출력 장치)
- 3가지 프로젝트: 문 열림 경보기 / 자동 조명 / 온도 제어
- 교과서 프로젝트: 마이크로프로세서로 문 열림 경보기 만들기 (p48)
- 배경: 0xeef0f5
- 카메라: PerspectiveCamera(FOV 50) + OrbitControls
- 조명: AmbientLight(0.5) + DirectionalLight(0.6)

## 프로젝트 1: 문 열림 경보기 (교과서 프로젝트)

### 3D
- 문: BoxGeometry(1.5, 3, 0.1), 0x886644 (나무색), 힌지(회전축)로 열림/닫힘
- 자석 센서(리드 스위치): BoxGeometry(0.2, 0.1, 0.1) × 2, 0x888888
  - 문+문틀에 각각 부착
  - 문 닫힘: 두 자석 가까이 → 스위치 닫힘 (신호 0)
  - 문 열림: 두 자석 멀어짐 → 스위치 열림 (신호 1)
- 마이크로프로세서(아두이노): BoxGeometry(1, 0.6, 0.1), 0x00aa44 (초록 PCB)
  - "Arduino" 라벨
  - LED 여러 개, 핀 헤더
- 부저(스피커): CylinderGeometry(0.3, 0.3, 0.1), 0x111111
  - 경보 시: 빨간 글로우 + "🔔" 라벨

### 동작 시뮬레이션
- 문 닫힘: 센서 신호 0 → 아두이노 → 부저 OFF, LED 초록
- 문 열림 (슬라이더/클릭으로 문 열기): 센서 신호 1 → 아두이노 → 부저 ON! LED 빨강 깜빡
- 데이터 흐름: 센서→전선→아두이노→전선→부저 (파티클로 신호 전달 시각화)

### 순서도 (하단)
- 시작 → 센서 값 읽기 → 문 열림? → 예: 경보 → 아니오: 대기 → 반복
- 프로그래밍 로직 시각화

## 프로젝트 2: 자동 조명 (광 센서)

### 3D
- 방: BoxGeometry(5, 3, 5) 반투명 벽
- 광 센서(CdS): SphereGeometry(r=0.15), 0xddcc44, 창문 근처
- 마이크로프로세서: 위와 동일
- LED 조명: SphereGeometry(r=0.3) emissive, 천장에 부착

### 동작
- 밝을 때 (광 센서 값 높음): LED OFF (절전)
- 어두울 때 (광 센서 값 낮음): LED ON (자동 점등)
- 밝기 슬라이더: AmbientLight 세기 조절 → 자동 반응

## 프로젝트 3: 온도 제어 (온도 센서 + 팬)

### 3D
- 온도 센서: BoxGeometry(0.2, 0.5, 0.1), 0x4488aa
- 팬(선풍기): CylinderGeometry(축) + PlaneGeometry(날개 3개), 0xdddddd
- 디스플레이: BoxGeometry(0.5, 0.3, 0.05), LED 숫자로 온도 표시

### 동작
- 온도 < 25°C: 팬 OFF
- 온도 ≥ 25°C: 팬 ON (날개 회전 애니메이션)
- 온도 슬라이더: 15~40°C → 팬 자동 제어

## IoT 네트워크 시각화
- "IoT 보기" 토글:
  - 각 장치에서 Wi-Fi 신호(동심원 파동) 발생
  - 클라우드(SphereGeometry, 0xddddff): 데이터 수집/분석
  - 스마트폰: BoxGeometry, 원격 모니터링/제어
  - "센서 → 마이크로프로세서 → 인터넷 → 클라우드 → 스마트폰"

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "프로젝트" 드롭다운: 문 열림 경보기 / 자동 조명 / 온도 제어
- "문 열기" 토글 (프로젝트 1): ON/OFF
- "밝기" 슬라이더 (프로젝트 2): 0~100%
- "온도" 슬라이더 (프로젝트 3): 15~40°C
- "데이터 흐름 표시" 토글: ON/OFF (기본값 ON)
- "순서도 표시" 토글: ON/OFF
- "IoT 네트워크" 토글: ON/OFF
- "초기화" 버튼

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널로 표시:

```
🌐 IoT와 마이크로프로세서
━━━━━━━━━━━━━━━━
프로젝트: [문 경보/자동 조명/온도 제어]

IoT 구조:
  센서(입력) → 마이크로프로세서(처리)
  → 액추에이터(출력)

현재 상태:
  센서 입력: [값]
  처리 결과: [조건 판단]
  출력: [ON/OFF/값]

프로젝트 1 (문 경보):
  리드 스위치 → 아두이노 → 부저
  문 열림 감지 → 경보!

IoT: 기기들이 인터넷으로 연결
  → 원격 모니터링/제어 가능
```

- 탐구 질문:
  1. "문 열림 경보기의 센서→처리→출력 과정을 순서도로 그려 보세요."
  2. "자동 조명 시스템에서 광 센서 대신 적외선 센서를 사용하면 어떤 기능이 가능할까요?"
  3. "IoT 기술이 우리 생활을 어떻게 변화시키고 있는지 3가지 예를 들어 보세요."
- 실생활 예시: 스마트 홈(자동 조명/온도), 웨어러블(스마트워치), 스마트 팜, 자율주행
- 교과서 연계: "전자기와 양자 I편 p37~48 — IoT, 마이크로프로세서, 프로젝트"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원 (OrbitControls 기본 지원)
- 프로젝트 전환 시 이전 객체 dispose() + 새 객체 생성
- gui.controllersRecursive().forEach(c => c.updateDisplay()) 동기화
- 성능: requestAnimationFrame 루프
- 파일명: emq_12_iot_microprocessor.html

너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 통합과학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 통합과학1
- 단원: III. 시스템과 상호작용 > 2. 역학 시스템 > 중력의 작용
- 토픽: 수평 방향 투사체 운동
- 대상: 고등학교 1학년
- 핵심 개념: 수평 방향으로 던진 물체는 수평 방향으로는 등속 운동, 연직 방향으로는 자유 낙하 운동(가속도 운동)을 한다. 같은 높이에서 자유 낙하 하는 물체와 수평 방향으로 던진 물체는 동시에 바닥에 도달한다. 수평 속력이 빠를수록 수평 이동 거리가 커진다. 두 운동이 합쳐져 포물선을 그린다.

## 씬 구성
- 화면을 좌우 분할:
  - 좌측(65%): 3D 실험 장면 (동시 낙하 장치)
  - 우측(35%): 운동 분석 그래프 패널
- 좌측 3D 장면:
  - 배경: 연한 하늘 그라데이션
  - 바닥: PlaneGeometry(15, 10) + GridHelper(15, 30, 0xaaaaaa)
  - 높이 눈금: Y축 좌측에 세로 눈금 (0~2m, 0.2m 간격)
  - 수평 거리 눈금: X축 바닥에 가로 눈금 (0~3m, 0.5m 간격)
  - 동시 낙하 장치 (교과서 탐구 p126~128):
    - 발사대: BoxGeometry(1, 0.1, 0.5), 0x555555, 높이 ~1.5m에 설치
    - 공 A (자유 낙하): SphereGeometry(r=0.1, 32, 32), MeshStandardMaterial(0xff4444 빨강, roughness 0.4, metalness 0.2), 발사대 끝에 배치, "A" 라벨
    - 공 B (수평 투사): SphereGeometry(r=0.1, 32, 32), MeshStandardMaterial(0x4488ff 파랑, roughness 0.4, metalness 0.2), 발사대 끝에 배치, "B" 라벨
  - 눈금판: PlaneGeometry(3, 2), MeshBasicMaterial(0xffffff, opacity 0.8), 공 뒤에 세로로 세움
    - 눈금: 0.1m 간격 수평선(LineGeometry, 0xcccccc) + 0.5m 간격 굵은 수평선(0x888888)
    - 높이 라벨: "0.0m", "0.5m", "1.0m", "1.5m" (Sprite 또는 CSS2DRenderer)
    - 교과서 탐구 p126~128 "눈금판(또는 모눈종이)" 재현
  - 공 잔상: 0.05초 간격, SphereGeometry(geometry 공유, Material clone, opacity 0.1), 최대 30개/공
    - 잔상으로 궤적 시각화: A는 수직 직선, B는 포물선
  - 중력 화살표: 각 공에서 아래 방향 ArrowHelper(0xff0000, 길이 0.3), "g" 라벨
- 카메라: PerspectiveCamera(FOV 50) + OrbitControls, 초기 위치 (3, 1.5, 5), lookAt(1.5, 0.5, 0)
- 조명: AmbientLight(0.5) + DirectionalLight(0.6, position(5, 8, 5)) + 그림자 활성화

## 운동 시뮬레이션

### 동시 발사 애니메이션
1. "발사" 버튼 클릭
2. 동시에:
   - 공 A: 수직 자유 낙하 (x=0 고정, y = y₀ - ½gt²)
   - 공 B: 수평 투사 (x = v₀ₓ × t, y = y₀ - ½gt²)
3. 핵심 시각화:
   - **매 순간 두 공의 y좌표(높이)가 동일**함을 수평 점선으로 연결
   - 수평 점선: LineDashedMaterial, 0x888888, 두 공 사이를 매 프레임 연결
   - 연직 방향 동시 낙하 강조: "같은 시간에 같은 높이!" 라벨
4. 궤적 표시:
   - 공 A: 수직 직선 궤적 (빨간 점선)
   - 공 B: 포물선 궤적 (파란 점선)
   - 궤적에 0.1초 간격으로 잔상 점 표시
5. 바닥 동시 도달: "동시에 도달!" 라벨 + 초록 플래시

### 운동 분해 표시 (교과서 그림 III-16, p128)
- 공 B의 운동을 수평/연직 성분으로 분해:
  - 수평 성분 (등속): X축 방향 파란 화살표 (길이 일정)
  - 연직 성분 (가속): Y축 방향 빨간 화살표 (길이 증가)
  - 합성 속도: 초록 화살표 (포물선 접선 방향)
- 각 화살표에 속도값 라벨: "vₓ = [값] m/s", "vᵧ = [값] m/s"

## 우측 그래프 패널

### 수평 거리-시간 그래프
- 공 A: x = 0 (수평선, 움직이지 않음)
- 공 B: x = v₀ₓ × t (직선, 기울기 = 초기 수평 속력)

### 높이-시간 그래프
- 공 A와 공 B: **동일한 곡선** (y = h - ½gt², 포물선)
- 두 곡선이 완전히 겹치는 것을 강조

### 속력-시간 그래프
- 공 A: 수직 속력만 = gt (직선)
- 공 B: 합성 속력 = √(v₀ₓ² + (gt)²) (곡선)

탭으로 3가지 그래프 전환.

## 수평 속력 변화 실험 (교과서 그림 III-17, p129)
- "다중 발사" 모드: 수평 속력이 다른 공 3개를 동시 발사
  - 공 B1: v₀ₓ = 2 m/s (짧은 포물선)
  - 공 B2: v₀ₓ = 4 m/s (중간 포물선)
  - 공 B3: v₀ₓ = 6 m/s (긴 포물선)
- 3개 모두 같은 시간에 바닥 도달 (연직 방향 동일)
- 수평 이동 거리만 다름
- "수평 속력이 빠를수록 더 멀리 날아간다" 라벨

## 인터랙션 (GUI 패널)
- "수평 속력" 슬라이더: 0 ~ 10 (기본값 3), 단위 m/s, 0.5 단계
- "높이" 슬라이더: 0.5 ~ 5.0 (기본값 1.5), 단위 m
- "발사" 버튼
- "다중 발사 모드" 토글: ON/OFF
- "속도 분해 표시" 토글: ON/OFF (기본값 ON) — 수평/연직 화살표
- "높이 연결선" 토글: ON/OFF (기본값 ON) — 두 공 사이 수평 점선
- "슬로모션" 슬라이더: 0.1x ~ 2x
- "그래프 유형" 드롭다운: 수평 거리-시간 / 높이-시간 / 속력-시간
- "초기화" 버튼

## 수식 패널
```
📐 수평 방향 투사체 운동
━━━━━━━━━━━━━━━━━━
수평 속력:     v₀ₓ = [값] m/s
높이:          h = [값] m
경과 시간:     t = [값] s

수평 방향 (등속 운동):
  x = v₀ₓ × t = [값] m

연직 방향 (자유 낙하):
  y = h - ½gt² = [값] m
  vᵧ = gt = [값] m/s

핵심: 연직 방향 운동은 자유 낙하와 동일!
→ 수평으로 던진 물체와 자유 낙하 물체는 동시에 도달
```

## 교육적 요소
- 핵심 원리: "수평 방향으로 던진 물체는 수평으로는 등속 운동, 연직으로는 자유 낙하를 합니다. 두 운동이 독립적으로 합쳐져 포물선 궤적이 됩니다."
- 탐구 질문:
  1. "공 A와 B가 동시에 바닥에 도달하는 까닭은?" (답: 연직 방향 가속도가 동일)
  2. "수평 속력을 2배로 하면 수평 이동 거리는?" (답: 2배)
  3. "지구보다 중력이 2배인 곳에서 같은 실험을 하면?" (답: 낙하 시간 짧아짐, 수평 이동 거리 감소)
- 교과서 연계: "미래엔 통합과학1 III편 p126~129 — 자유 낙하와 수평 투사 비교(그림 III-16, III-17)"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원 (OrbitControls 기본 지원), 모바일에서 lil-gui 기본 접힘 상태
- 물리: Clock.getDelta() × 슬로모션, 공 A: y = y0 - 0.5*g*t^2 (x=0 고정), 공 B: y = y0 - 0.5*g*t^2, x = v0x*t (독립 운동)
- 궤적: Line + BufferGeometry, 매 프레임 공 위치를 점으로 추가, 궤적 색상 = 공 색상, linewidth 2
- 높이 연결선: LineDashedMaterial(color 0x888888, dashSize 0.05, gapSize 0.05), 양쪽 공의 현재 Y좌표를 매 프레임 Line으로 연결
- 속도 분해 화살표: ArrowHelper 3개/공B (수평 vx=파랑, 연직 vy=빨강, 합성 v=초록), 매 프레임 방향·길이 업데이트
- 잔상: geometry 공유, material.opacity 0.1, 최대 30개/공, 오래된 것 dispose()
- 그래프: Canvas 2D API, 3종 그래프 탭 전환 (clearRect + 재그리기)
- 바닥 충돌: y <= 0 감지, 정지 + "도달!" 라벨
- 성능: requestAnimationFrame, 총 mesh 최대 ~80개 (공 2 + 잔상 60 + 화살표 6 + 기타)
- 파일명: projectile_motion_sim.html

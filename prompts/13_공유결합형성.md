너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 통합과학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 통합과학1
- 단원: II. 물질과 규칙성 > 1. 원소의 생성과 규칙성 > 이온 결합과 공유 결합
- 토픽: 공유 결합 형성
- 대상: 고등학교 1학년
- 핵심 개념: 비금속 원소의 원자들이 전자쌍을 공유하며 형성하는 결합을 공유 결합이라고 한다. 수소는 원자가 전자 수가 1이므로 1개의 전자쌍을 공유하고, 산소는 원자가 전자 수가 6이므로 2개의 전자쌍을 공유한다. 공유 결합을 통해 각 원자는 18족 원소와 같은 안정한 전자 배치를 갖는다.

## 씬 구성
- 화면 중앙에 3D 분자 모형을 렌더링한다.
- 좌측에 분자 선택 패널, 우측에 전자 배치 비교 패널을 HTML overlay로 배치.
- 배경: 0x0a0e1e (짙은 남색)
- 조명: AmbientLight(0.35) + DirectionalLight(0.65) + PointLight(분자 중심, 0xffffff, 세기 1.0)
- 카메라: PerspectiveCamera(FOV 50) + OrbitControls (분자를 마우스로 360도 회전)

## 3가지 분자 모형 상세

### (1) 물 분자 H₂O (교과서 그림 II-22, p70)
- 산소 원자 보어 모형:
  - 원자핵: SphereGeometry(r=0.35), 0xff4444, "+8" 라벨
  - 1번 껍질: TorusGeometry(r=0.7), 전자 2개(파랑 구 r=0.06)
  - 2번 껍질: TorusGeometry(r=1.3), 전자 6개(파랑) + 빈 자리 2개(점선 원)
- 수소 원자 보어 모형 (2개):
  - 원자핵: SphereGeometry(r=0.2), 0xff4444, "+1" 라벨
  - 1번 껍질: TorusGeometry(r=0.5), 전자 1개(파랑)
- 공유 결합 애니메이션:
  1. O 원자 중앙, H 원자 2개가 좌우에 배치 (간격 4.0)
  2. H와 O가 서로 접근 (4.0 → 1.8, 1.5초)
  3. 접근하면서 전자 궤도가 겹치기 시작
  4. 결합 순간: 공유 전자쌍 형성 — 겹친 영역의 전자 2개(H에서 1개 + O에서 1개)가 노란색(0xffdd44)으로 변경, 두 원자 사이를 왕복하는 8자 궤도 애니메이션
  5. 각 H-O 결합에 공유 전자쌍 1쌍씩, 총 2쌍
  6. 결합 완료: "H₂O (물 분자)" 라벨 + 결합각 104.5° 호(Arc) 표시
- 전자 배치 확인:
  - O: 2 + 6 + 공유 전자 2개 = 2, 8 → 네온(Ne)과 동일 ✅
  - H: 1 + 공유 전자 1개 = 2 → 헬륨(He)과 동일 ✅
  - 각 원자 옆에 "18족과 같은 전자 배치 → 안정!" 라벨

### (2) 산소 분자 O₂ (교과서 그림 II-23, p70)
- 산소 원자 2개 보어 모형 (동일한 구조)
- 공유 결합 애니메이션:
  1. O 원자 2개가 좌우에 배치 (간격 4.0)
  2. 접근하며 2번 껍질 전자가 겹침
  3. 결합 순간: 공유 전자쌍 2쌍 형성 (이중 결합)
     - 공유 전자 4개(각 O에서 2개씩)가 노란색, 두 원자 사이 이중 궤도
  4. "O₂ (산소 분자) — 이중 결합" 라벨
- 전자 배치:
  - 각 O: 2 + 6 + 공유 전자 2개 = 2, 8 → Ne과 동일 ✅
- 교과서: "산소 분자(O₂)는 2개의 산소 원자가 서로 2개의 전자쌍을 공유하며 결합을 형성한다" (p70)

### (3) 플루오린화 수소 HF (교과서 탐구 p71)
- F 원자 보어 모형:
  - 원자핵: r=0.3, "+9"
  - 1번 껍질: 전자 2개, 2번 껍질: 전자 7개 + 빈 자리 1개
- H 원자 보어 모형: "+1", 전자 1개
- 결합: 공유 전자쌍 1쌍 (단일 결합)
- F: 2, 7 + 공유 1 = 2, 8 → Ne ✅
- H: 1 + 공유 1 = 2 → He ✅

## 분자 모형 전환 (공-막대 모형)
- "보어 모형 / 공-막대 모형" 토글:
  - 보어 모형: 위에서 설명한 전자 껍질 + 전자 표시
  - 공-막대 모형: 원자를 큰 구(색상 구분)로, 결합을 막대(CylinderGeometry)로 표시
    - O: 빨간 구(r=0.5, 0xff2222)
    - H: 흰 구(r=0.3, 0xffffff)
    - F: 연초록 구(r=0.45, 0x88ff88)
    - 단일 결합: CylinderGeometry(r=0.06) 1개
    - 이중 결합: CylinderGeometry(r=0.04) 2개 나란히
  - 결합 길이, 결합각 라벨 표시

## 공유 전자쌍 vs 비공유 전자쌍 시각화
- 공유 전자쌍: 노란색(0xffdd44), 두 원자 사이에 위치
- 비공유 전자쌍: 파란색(0x4488cc), 원자 바깥쪽에 위치
- 전자쌍 표시 토글: ON 시 각 전자쌍에 라벨("공유", "비공유") 표시
- H₂O 예시: 공유 전자쌍 2쌍 + O의 비공유 전자쌍 2쌍

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "분자 선택" 드롭다운: H₂O (물) / O₂ (산소) / HF (플루오린화 수소)
- "표시 모드" 드롭다운: 보어 원자 모형 / 공-막대 모형
- "결합 과정 재생" 버튼: 공유 결합 형성 애니메이션 재생
- "전자쌍 표시" 토글: ON/OFF (기본값 ON)
- "18족 비교" 토글: ON 시 해당 원자의 전자 배치와 18족 원소의 전자 배치를 나란히 비교
- "공전 속도" 슬라이더: 0x ~ 5x (기본값 1x)
- "초기화" 버튼

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널로 표시:

```
🔗 공유 결합 형성
━━━━━━━━━━━━━━━
선택 분자:       [H₂O / O₂ / HF]
결합 유형:       [단일 결합 / 이중 결합]
공유 전자쌍:     [n]쌍
비공유 전자쌍:   [n]쌍

전자 배치 변화:
  [원자1]: [변화 전] → [변화 후] ([18족 원소]과 동일 ✅)
  [원자2]: [변화 전] → [변화 후] ([18족 원소]과 동일 ✅)

원리: 비금속 원소의 원자들이 전자쌍을
공유하여 18족 원소와 같은 안정한
전자 배치를 갖는다.
```

## 교육적 요소
화면 상단에 토글 가능한 "📖 학습 가이드" 패널:
- 핵심 원리: "비금속 원소의 원자들은 전자쌍을 공유하여 18족 원소와 같은 안정한 전자 배치를 가집니다. 공유하는 전자쌍의 수에 따라 단일 결합, 이중 결합으로 구분합니다."
- 탐구 질문:
  1. "물 분자에서 산소는 수소와 몇 쌍의 전자를 공유하나요?"
  2. "산소 분자(O₂)가 이중 결합인 까닭은 무엇인가요?"
  3. "공유 결합과 이온 결합의 차이점은 무엇인가요? (힌트: 전자를 주고받기 vs 공유)"
- 실생활 예시: 물(H₂O), 산소(O₂ — 호흡), 이산화 탄소(CO₂), 설탕(C₁₂H₂₂O₁₁), 뷰테인(C₄H₁₀ — 가스레인지)
- 교과서 연계: "미래엔 통합과학1 II편 p70~71 — 공유 결합, 물 분자(그림 II-22), 산소 분자(그림 II-23), 탐구(공유 결합 모형)"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원
- 전자 공전: requestAnimationFrame, TorusGeometry 궤도 위 원운동
- 공유 전자쌍: 두 원자 사이 8자 궤도 또는 타원 궤도
- 분자 전환 시 이전 객체 dispose()
- 성능: requestAnimationFrame 루프, 전자 최대 18개
- 파일명: covalent_bonding_formation_sim.html

너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 물리학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 역학과 에너지
- 단원: I. 시공간과 운동 > 06. 등가 원리와 시공간의 휨
- 토픽: 시공간의 휨과 중력 렌즈
- 대상: 고등학교 (진로 선택과목)
- 핵심 개념: 아인슈타인의 일반 상대성 이론에 따르면 질량이 있는 물체는 주위의 시공간을 휘게 한다. 질량이 클수록 시공간의 휨이 커진다. 중력은 시공간의 휨에 의한 효과이다. 중력에 의해 빛의 경로가 휘어지며(중력 렌즈), 강한 중력장에서는 시간이 느리게 흐른다(중력 시간 지연). 블랙홀은 시공간을 극도로 휘게 하여 빛조차 빠져나올 수 없다.

## 씬 구성
- 화면 중앙: 3D 시공간 격자(고무 시트 비유) + 질량에 의한 휨 시각화
- 3가지 모드: 시공간 휨 / 중력 렌즈 / 중력 시간 지연
- 배경: 0x000010 (우주)
- 카메라: PerspectiveCamera(FOV 50) + OrbitControls (비스듬히 위에서)
- 조명: AmbientLight(0.2) + DirectionalLight(0.5)

## 모드 1: 시공간의 휨 (고무 시트 비유)

### 3D 격자 시트
- PlaneGeometry(20, 20, 40, 40) — 세밀한 격자
- MeshStandardMaterial(wireframe: true), 0x44ff88 (초록 격자)
- 초기: 평평한 격자 (중력 없는 시공간)

### 질량에 의한 변형
- 격자 중앙에 질량 M 배치: SphereGeometry(r), 0xffcc44 (노란 구)
- 격자가 질량 주위에서 아래로 움푹 들어감 (y좌표 변형)
  - 변형량: Δy = -k·M / (r² + ε) (r: 질량까지 거리)
  - 질량 클수록 더 깊이 들어감
  - 가까울수록 더 많이 휨
- BufferGeometry의 position 속성을 실시간 수정
- 질량 슬라이더: 질량 증가 → 움푹 들어가는 정도 증가
- "질량이 시공간을 휘게 한다" 라벨

### 두 번째 물체 (행성)
- 작은 구(SphereGeometry r=0.2, 0x2266aa)를 격자 위에 놓으면
- 휘어진 격자를 따라 큰 질량 쪽으로 굴러감 → "중력"의 기하학적 해석
- 궤적: 직선이 아닌 곡선 → 시공간의 휨 때문
- "물체는 휘어진 시공간에서 가장 자연스러운 경로(측지선)를 따라 운동한다"

### 블랙홀 극한
- 질량을 극단적으로 키우면: 격자가 매우 깊은 구멍 형성
- "시공간이 극도로 휘어져 빛조차 빠져나올 수 없다 → 블랙홀"
- 사건의 지평선 반지름에 원(RingGeometry) 표시

## 모드 2: 중력 렌즈

### 3D
- 먼 배경 별: Points(500개), 0xffffff, 랜덤 분포
- 중간에 거대 질량(은하단): SphereGeometry(r=1), 0xffaa44 반투명
- 배경 별빛이 거대 질량 주위에서 휘어져 관측자에게 도달

### 중력 렌즈 효과 시각화
- 별빛 경로: TubeGeometry 곡선, 0xffdd44
  - 질량 없을 때: 직선
  - 질량 있을 때: 곡선 (질량 주위에서 휘어짐)
- 아인슈타인 고리: 특정 배치에서 배경 별이 고리 모양으로 보임
  - RingGeometry, 0xffee88, 거대 질량 주위에
- 상의 왜곡: 배경 은하가 호(arc) 모양으로 늘어나 보임
- "중력에 의해 빛의 경로가 휘어진다 → 중력 렌즈" 라벨
- 교과서: 1919년 개기 일식 때 별빛이 태양 근처에서 휘어진 것 관측 (에딩턴)

## 모드 3: 중력 시간 지연

### 3D
- 지구(또는 블랙홀) 근처에 2개의 시계 배치
  - 시계 A: 지표면(강한 중력) — 디지털 시계, 0xff4444
  - 시계 B: 높은 궤도(약한 중력) — 디지털 시계, 0x4444ff
- 두 시계가 동시에 시작하지만:
  - 강한 중력 → 시간이 느리게 흐름 (A가 느림)
  - 약한 중력 → 시간이 빠르게 흐름 (B가 빠름)
- 시간 차이를 실시간으로 표시

### GPS 보정 응용
- GPS 위성(궤도 20,200km): 약한 중력 → 시계가 하루 약 45μs 빠름
- 보정하지 않으면: 하루 ~10km 위치 오차!
- "GPS가 정확하려면 일반 상대성 이론 보정이 필수" 라벨

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "모드" 드롭다운: 시공간 휨 / 중력 렌즈 / 시간 지연
- "질량 M" 슬라이더: 1~100 (상대값, 기본값 10)
- "행성 놓기" 버튼 (모드 1): 격자 위에 작은 물체 놓기
- "렌즈 질량" 슬라이더 (모드 2): 1~50
- "고도 차이" 슬라이더 (모드 3): 0~50,000km
- "격자 표시" 토글: ON/OFF (기본값 ON)
- "빛 경로 표시" 토글: ON/OFF
- "초기화" 버튼

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널로 표시:

```
🌌 시공간의 휨과 중력 렌즈
━━━━━━━━━━━━━━━━━
모드: [시공간 휨/중력 렌즈/시간 지연]
질량 M = [값] (상대값)

일반 상대성 이론 (아인슈타인, 1915):
  질량이 시공간을 휘게 한다
  → 중력 = 시공간 휨의 효과

시공간 휨:
  질량 클수록 → 더 많이 휨
  블랙홀 → 극도로 휨

중력 렌즈:
  빛의 경로가 중력에 의해 휨
  → 배경 천체의 상이 왜곡

중력 시간 지연:
  강한 중력 → 시간이 느리게
  GPS 보정: 하루 ~45μs 차이
```

## 교육적 요소
화면 상단에 토글 가능한 "📖 학습 가이드" 패널:
- 핵심 원리: "질량이 있는 물체는 주위의 시공간을 휘게 합니다. 중력은 시공간의 휨에 의한 효과이며, 빛의 경로도 휘어집니다(중력 렌즈). 강한 중력장에서는 시간이 느리게 흐릅니다(중력 시간 지연)."
- 탐구 질문:
  1. "고무 시트 비유에서 무거운 공을 놓으면 시트가 휘어지는 것이 시공간의 휨과 어떤 관계가 있나요?"
  2. "1919년 개기 일식 관측이 일반 상대성 이론을 입증한 까닭은 무엇인가요?"
  3. "GPS에서 일반 상대성 이론 보정이 필요한 까닭을 설명해 보세요."
- 실생활 예시: GPS 시간 보정, 중력파 검출(LIGO), 블랙홀 사진(EHT), 중력 렌즈 관측
- 교과서 연계: "역학과 에너지 I편 p52~59 — 등가 원리, 시공간의 휨, 중력 렌즈, 블랙홀"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원 (OrbitControls 기본 지원)
- 모드 전환 시 이전 객체 dispose() + 새 객체 생성
- 격자 변형: PlaneGeometry의 position attribute 직접 수정 (매 프레임 아님, 질량 변경 시)
- gui.controllersRecursive().forEach(c => c.updateDisplay()) 동기화
- 성능: requestAnimationFrame 루프, 격자 40×40=1600 정점
- 파일명: mech_12_spacetime_curvature.html

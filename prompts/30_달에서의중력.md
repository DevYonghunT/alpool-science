너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 통합과학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 통합과학1
- 단원: III. 시스템과 상호작용 > 2. 역학 시스템 > 중력의 작용
- 토픽: 달에서의 중력
- 대상: 고등학교 1학년
- 핵심 개념: 중력은 보통 지구가 물체를 끌어당기는 힘을 뜻하지만, 달처럼 질량이 있는 천체도 주위의 물체에 중력을 작용한다. 달 표면에서 중력은 지구의 약 1/6이므로 지구에서보다 물체가 느리게 떨어진다. 중력 가속도: 지구 g=9.8m/s², 달 g≈1.63m/s².

## 씬 구성
- 화면을 좌우 분할하여 2개의 독립적인 3D 씬을 렌더링한다.
- **구현 방식**: renderer 1개, `renderer.setScissor()`와 `renderer.setViewport()`로 좌/우 viewport 분할. 각 viewport에 별도의 Scene + Camera.
- 중앙 구분선: CSS border-right(1px solid 0x444444) 또는 Three.js Line

### 좌측 viewport: 지구 환경
- 배경: 하늘 그라데이션 (Scene.background = 그라데이션 텍스처)
  - 상단: 0x87CEEB (하늘색) → 하단: 0xE0F0FF (연한 하늘)
- 바닥:
  - PlaneGeometry(10, 10), MeshStandardMaterial(0x44aa44 풀밭 녹색, roughness 0.9)
  - GridHelper(10, 20, 0x338833, 0x338833) 바닥 위에 배치
- 높이 눈금 기둥:
  - CylinderGeometry(r=0.02, h=낙하 높이), 0x333333 (검정), 좌측 끝에 세로로 세움
  - 1m 간격 수평 눈금 막대(BoxGeometry 0.3×0.01×0.01) + 높이 라벨("0m", "1m", "2m", ...)
  - HTML overlay 라벨: Sprite 또는 CSS2DRenderer
- 환경 오브젝트(분위기):
  - 나무 1그루: CylinderGeometry(줄기 r=0.1, h=1.5, 0x8B4513) + SphereGeometry(잎 r=0.6, 0x228b22), 좌측 뒤에 배치
  - 구름 2개: SphereGeometry(r=0.3~0.5, 0xffffff, opacity 0.6), 상단 좌우에 느리게 이동
- 라벨: "🌍 지구" (상단 중앙, HTML overlay, 24px bold 흰색, 텍스트 그림자)
- 하위 라벨: "g = 9.8 m/s²" (18px, 연한 흰색)
- 조명: AmbientLight(0.5) + DirectionalLight(0.6, 좌상단에서, 그림자 활성화)
- 카메라: PerspectiveCamera(FOV 50), 초기 위치 (5, 3, 5), lookAt(0, 2, 0)

### 우측 viewport: 달 환경
- 배경: 우주 검정 (0x060608)
  - 별 파티클: Points(500개, BufferGeometry, 랜덤 위치, 0xffffff, 크기 0.5~2.0)
- 바닥:
  - PlaneGeometry(10, 10), MeshStandardMaterial(0x888888 회색, roughness 1.0)
  - 크레이터 표현:
    - 원형 홈 5~8개: CylinderGeometry(r=0.3~0.8, h=0.05, 32), 0x666666 (바닥보다 어둡게)
    - 크레이터 테두리: TorusGeometry(r=크레이터 반지름, tube=0.03), 0x999999 (바닥보다 밝게)
    - 랜덤 위치에 분산 배치
  - GridHelper(10, 20, 0x555555, 0x555555) — 지구보다 어둡게
- 높이 눈금: 지구와 동일 구조, 동일 높이
- 환경 오브젝트:
  - 달 표면 암석 3~5개: IcosahedronGeometry(r=0.1~0.3, detail=0), 0x777777
  - 먼 곳에 지구: SphereGeometry(r=0.3), 0x2266aa, 상단에 작게 배치 (지구에서 달을 바라보는 역방향)
- 라벨: "🌙 달" (상단 중앙, 24px bold 흰색)
- 하위 라벨: "g = 1.63 m/s² (지구의 약 1/6)" (18px)
- 조명: AmbientLight(0.15, 달의 어두운 환경) + DirectionalLight(0.8, 한쪽에서 강하게, 그림자 활성화) — 달의 극단적 명암 차이 표현
- 카메라: PerspectiveCamera(FOV 50), 지구 카메라와 동일 각도/거리

## 낙하 비교 시뮬레이션

### 낙하 물체
- 양쪽 동일 물체: SphereGeometry(r=0.2, 32, 32), MeshStandardMaterial(0xff4444 빨강, metalness 0.3, roughness 0.4)
- 각 공 위에 질량 라벨: "m = 1 kg" (Sprite 또는 CSS2DRenderer)
- 각 공 옆에 실시간 속력 라벨: "v = [값] m/s" (매 프레임 업데이트)

### 낙하 애니메이션 상세
1. "동시 낙하" 버튼 클릭
2. 양쪽 공이 동시에 같은 높이(설정값)에서 자유 낙하 시작
3. 물리 (requestAnimationFrame 기반):
   - deltaTime = clock.getDelta() × 슬로모션 배율
   - 지구: velocity_earth += 9.8 × dt, y_earth -= velocity_earth × dt
   - 달: velocity_moon += 1.63 × dt, y_moon -= velocity_moon × dt
4. 잔상 표시 (0.15초 간격):
   - 지구: 빨간 반투명 구(opacity 0.1) — 간격 좁음 (빠르게 떨어짐)
   - 달: 빨간 반투명 구(opacity 0.1) — 간격 넓음 (느리게 떨어짐)
   - 잔상 최대 30개/공, 오래된 것부터 fade out 후 제거
5. 지구 공 먼저 도달:
   - 바닥 충돌 시: 공 정지 + 바운스 효과(y 약간 튀어올랐다 정지, 0.3초)
   - "✅ 지구 도달!" 초록 라벨 + 도달 시간 표시: "t = [값]초"
   - 달 공의 현재 높이를 빨간 수평 점선으로 표시: "달 공: 아직 [높이]m"
6. 달 공 나중 도달:
   - 동일한 바운스 효과
   - "✅ 달 도달!" 초록 라벨 + 도달 시간 표시
   - **시간 비교 표시**: "지구: [t₁]초 vs 달: [t₂]초 (약 [배율]배 차이)"
7. 낙하 완료 후 비교 요약 패널 (하단 HTML overlay):
   ```
   ⏱️ 낙하 시간 비교
   지구: [t₁]초 | 달: [t₂]초
   달에서 약 [t₂/t₁]배 느리게 떨어짐
   (√(지구g/달g) = √(9.8/1.63) ≈ 2.45배)
   ```

### 속력-시간 비교 그래프 (하단 좌측)
- Canvas 2D 렌더링 (HTML overlay 내 <canvas>)
- X축: 시간 (0~최대 낙하 시간 × 1.2), Y축: 속력 (0~최대 속력 × 1.2)
- 지구 곡선: 빨간 직선 (기울기 9.8), 라벨 "지구 (기울기 = 9.8 m/s²)"
- 달 곡선: 파란 직선 (기울기 1.63), 라벨 "달 (기울기 = 1.63 m/s²)"
- 현재 시점 표시: 양쪽 곡선 위에 움직이는 점
- 지구 도달 시: 지구 곡선에 수직 점선 + "도달" 마크

## 점프 비교 모드

### 3D 사람 실루엣
- 단순 인체: 
  - 머리: SphereGeometry(r=0.15), 0xffcc99
  - 몸통: BoxGeometry(0.3, 0.5, 0.15), 0x4488cc (파란 옷)
  - 팔: CylinderGeometry(r=0.04, h=0.4) × 2, 0xffcc99
  - 다리: CylinderGeometry(r=0.05, h=0.5) × 2, 0x334466 (바지)
- 양쪽 환경에 동일한 사람 배치

### 점프 애니메이션
1. "점프!" 버튼 클릭
2. 양쪽 사람이 동시에 같은 초기 속도(v₀ = 3.13 m/s)로 점프
3. 물리:
   - 지구: y = v₀t - ½(9.8)t², 최고점 = v₀²/(2g) = 3.13²/(2×9.8) ≈ 0.5m
   - 달: y = v₀t - ½(1.63)t², 최고점 = 3.13²/(2×1.63) ≈ 3.0m
4. 점프 중 애니메이션:
   - 상승: 팔이 위로 올라감 (rotation 변화)
   - 최고점: 잠시 정지 느낌
   - 하강: 팔 내려옴
   - 착지: 다리 약간 구부러짐
5. 최고점 표시:
   - 수평 점선 + 높이 라벨: "최고점: 0.5m" (지구) vs "최고점: 3.0m" (달)
   - "달에서 약 6배 높이 점프!" 강조 라벨 (노란색 배경, 큰 폰트)
6. 체공 시간 비교:
   - 지구: 2v₀/g = 2×3.13/9.8 ≈ 0.64초
   - 달: 2v₀/g = 2×3.13/1.63 ≈ 3.84초
   - "체공 시간: 0.64초 vs 3.84초 (약 6배)"

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "실험 모드" 드롭다운: 낙하 비교 / 점프 비교
- 낙하 모드:
  - "낙하 높이" 슬라이더: 1 ~ 20 (기본값 5), 단위 m, 1 단계
  - "동시 낙하" 버튼: 낙하 애니메이션 시작
  - "물체 질량" 슬라이더: 0.1 ~ 100 (기본값 1), 단위 kg
    - 질량이 바뀌어도 낙하 시간 동일(핵심: 질량 무관) → "질량에 관계없이 동시에 떨어진다!" 라벨
- 점프 모드:
  - "초기 속도" 슬라이더: 1.0 ~ 5.0 (기본값 3.13), 단위 m/s, 0.1 단계
  - "점프!" 버튼
- 공통:
  - "슬로모션" 슬라이더: 0.1x ~ 3x (기본값 1x), 0.1 단계
  - "잔상 표시" 토글: ON/OFF (기본값 ON)
  - "속력 라벨" 토글: ON/OFF (기본값 ON) — 각 공/사람 옆 실시간 속력
  - "비교 천체" 드롭다운: 지구+달(기본) / 지구+화성(g=3.72m/s²) / 지구+목성(g=24.79m/s²) / 지구+무중력(g=0)
    - 천체 변경 시 우측 환경 배경/바닥/라벨 모두 변경
    - 화성: 배경 0x331111 (붉은 우주), 바닥 0xaa4422 (붉은 토양)
    - 목성: 배경 0x111122, 바닥 줄무늬 0xdd9944
    - 무중력: 배경 0x000000, 바닥 없음(공이 안 떨어짐)
  - "초기화" 버튼: 공/사람을 초기 위치로 복원

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널(배경 rgba(0,0,0,0.7), 둥근 모서리 12px, padding 20px)로 표시:

```
🌙 달에서의 중력
━━━━━━━━━━━━━━━
실험 모드:     [낙하 / 점프]
비교 천체:     [지구 + 달/화성/목성]
경과 시간:     [t] 초

[낙하 모드]
         지구          달
g:       9.8 m/s²     1.63 m/s²
속력:    [v₁] m/s     [v₂] m/s
높이:    [y₁] m       [y₂] m
도달:    [t₁] 초      [t₂] 초

낙하 시간 비율: t_달/t_지구 = √(g_지구/g_달) ≈ 2.45

[점프 모드]
초기 속도:     v₀ = [값] m/s
최고점(지구):  h = v₀²/(2×9.8) = [값] m
최고점(달):    h = v₀²/(2×1.63) = [값] m
비율:          h_달/h_지구 ≈ 6.0배
```
모든 값은 실시간 업데이트.

## 교육적 요소
화면 상단에 토글 가능한 "📖 학습 가이드" 패널:
- 핵심 원리: "달처럼 질량이 있는 천체도 주위의 물체에 중력을 작용합니다. 달의 중력은 지구의 약 1/6이므로 물체가 느리게 떨어지고, 같은 힘으로 약 6배 높이 점프할 수 있습니다."
- 탐구 질문:
  1. "달에서 물체를 떨어뜨리면 지구보다 느리게 떨어지는 까닭은 무엇인가요?" (답: 달의 중력 가속도가 지구의 약 1/6이므로)
  2. "아폴로 우주 비행사들이 달에서 크게 뛰어다닐 수 있었던 까닭은?" (답: 약한 중력 → 같은 힘으로 더 높이 점프)
  3. "목성에서 물체를 떨어뜨리면 어떻게 될까요?" (답: g=24.79m/s²로 지구보다 약 2.5배 빠르게 떨어짐)
  4. "질량이 다른 두 물체를 달에서 동시에 떨어뜨리면 어떻게 될까요?" (답: 공기 저항 없으므로 동시에 떨어짐 — 아폴로 15호 깃털-망치 실험)
  5. "무중력 상태(국제우주정거장)에서 물체를 놓으면 어떻게 될까요?" (답: 떨어지지 않고 떠 있음)
- 실생활 예시:
  - 아폴로 15호 깃털-망치 실험 (달에서 진공 상태 자유 낙하 실험)
  - 국제우주정거장(ISS)의 무중력 환경
  - 화성 탐사 로버 착륙 시 중력 차이 고려
  - SF 영화 속 달/화성 중력 표현
- 교과서 연계: "미래엔 통합과학1 III편 p131 — 달에서의 운동과 중력. '달 표면에서 중력은 지구의 약 1/6이므로 지구에서보다 물체가 느리게 떨어진다.'"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리 (좌/우 viewport 비율 유지)
- 모바일 터치 지원 (OrbitControls 기본 지원)
- **분할 화면 구현**: renderer.setScissor() + renderer.setViewport()로 좌/우 분할
  - render 루프에서: renderer.setScissorTest(true) → 좌측 viewport 렌더 → 우측 viewport 렌더
  - 각 viewport에 독립적인 Scene, Camera, OrbitControls
- 물리: Clock.getDelta() × 슬로모션 배율, y += velocity × dt, velocity += g × dt
- 잔상: SphereGeometry 재사용(geometry 1개 공유), Material만 clone + opacity 설정, 최대 30개
- 속력-시간 그래프: Canvas 2D API (별도 <canvas> 요소, position absolute)
- 사람 실루엣: Group으로 머리+몸통+팔+다리 조립, 점프 시 rotation 애니메이션
- 천체 변경 시: Scene.background 교체 + 바닥 material.color 교체 + 라벨 텍스트 교체
- 성능: requestAnimationFrame 루프, 잔상 최대 30개×2 = 60개 mesh
- 파일명: moon_gravity_comparison_sim.html

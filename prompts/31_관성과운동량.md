너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 통합과학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 통합과학1
- 단원: III. 시스템과 상호작용 > 2. 역학 시스템 > 역학 시스템과 안전
- 토픽: 관성과 운동량
- 대상: 고등학교 1학년
- 핵심 개념: 관성이란 물체가 운동 상태를 유지하려는 성질이다. 외부에서 힘이 작용하지 않으면 정지한 물체는 계속 정지하고, 운동하는 물체는 계속 등속 운동을 유지한다. 운동량 = 질량 × 속도 (단위: kg·m/s). 질량이 크고 속도가 빠를수록 운동량이 크다. 정지한 물체는 속도가 0이므로 운동량도 0이다.

## 씬 구성
- 2탭 구성: 관성 체험 / 운동량 비교
- 배경: 0xeeeeff (밝은 실내)
- 조명: AmbientLight(0.5) + DirectionalLight(0.6, 좌상단) + 그림자 활성화
- 카메라: PerspectiveCamera(FOV 50) + OrbitControls

## 탭 1: 관성 체험 (교과서 그림 III-22, p132)

### 3D 버스 모형
- 버스 본체: BoxGeometry(4, 1.8, 1.5), MeshStandardMaterial(0x4488cc 파란색, roughness 0.6)
  - 앞유리: PlaneGeometry(1.4, 0.8), MeshPhysicalMaterial(0xaaddff, transmission 0.7, roughness 0.05), 버스 전면 상부에 부착
  - 측면 창문: PlaneGeometry(0.6, 0.5) × 4개, MeshPhysicalMaterial(0xaaddff, transmission 0.6), 양쪽 측면에 등간격 배치
  - 문: BoxGeometry(0.7, 1.2, 0.05), 0x3377aa (짙은 파랑), 우측 전면에 배치
  - 범퍼: BoxGeometry(0.1, 0.3, 1.5), 0x333333 (검정), 전면·후면 하단
- 바퀴: CylinderGeometry(r=0.3, h=0.15, 16) × 4개, 0x222222
  - 바퀴 회전 애니메이션: 버스 이동 시 rotation.z 변화 (속도에 비례)
  - 바퀴 허브: CylinderGeometry(r=0.1, h=0.16), 0x888888 (은색)
- 바닥(도로): PlaneGeometry(30, 5), MeshStandardMaterial(0x555555 아스팔트)
  - 도로 중앙선: BoxGeometry(0.8, 0.02, 0.1) × 10개, 0xffffff, 등간격 배치 (점선)
  - 도로 이동 시 중앙선이 역방향으로 이동 → 버스가 달리는 느낌
- 배경 건물: BoxGeometry(2, 3, 1) × 3~4개, 0xccbbaa (베이지), 도로 양쪽 뒤에 배치

### 승객 3D 모형 (버스 내부)
- 좌석: BoxGeometry(0.4, 0.3, 0.4), 0xcc6644 (갈색), 2열 × 2개 = 4석
- 승객: 3명, 각각 좌석에 앉아있음
  - 머리: SphereGeometry(r=0.12, 16, 16), 0xffcc99 (피부색)
  - 상체: BoxGeometry(0.25, 0.35, 0.15), 0x44aacc (파란 옷) / 0xcc4444 (빨간 옷) / 0x44aa44 (초록 옷)
  - 하체: BoxGeometry(0.25, 0.3, 0.15), 0x334466 (바지), 좌석에 고정
  - 팔: CylinderGeometry(r=0.03, h=0.3) × 2, 0xffcc99, 상체 양옆
  - **상체와 머리만 기울어짐** (하체는 좌석에 고정 = 버스와 함께 움직임)
- 손잡이: CylinderGeometry(r=0.02, h=1.0), 0xdddddd (은색), 천장에서 내려옴 × 2개
  - 손잡이도 관성에 의해 기울어짐 (승객과 동일 방향)

### 급출발 애니메이션
1. 초기 상태: 버스 정지, 승객 직립, "정지 중..." 라벨
2. "급출발" 버튼 클릭:
   - 버스: x 방향 가속 출발 (0→5 m/s, 1초), 바퀴 회전 시작, 도로 중앙선 역방향 이동
   - 승객 상체: Z축 회전 -15° (뒤로 기울어짐, 0.3초 Tween)
   - 손잡이: Z축 회전 -10° (같은 방향)
   - 관성 화살표: 승객 머리 위에 "← 관성(정지 유지)" ArrowHelper(0x4488ff 파랑, 길이 0.5)
   - 라벨: "정지해 있으려는 관성 때문에 몸이 뒤로 쏠린다" (교과서 그림 III-22 좌측)
3. 등속 운동 전환: 승객 서서히 직립 복원 (0.5초), "등속 운동 중" 라벨

### 급정지 애니메이션
1. 등속 운동 상태에서 "급정지" 버튼 클릭:
   - 버스: 감속 (5→0 m/s, 0.5초), 바퀴 회전 정지
   - 승객 상체: Z축 회전 +20° (앞으로 기울어짐, 0.3초)
   - 손잡이: Z축 회전 +15°
   - 관성 화살표: "→ 관성(운동 유지)" ArrowHelper(0xff4444 빨강)
   - 라벨: "운동하려는 관성 때문에 몸이 앞으로 쏠린다" (교과서 그림 III-22 우측)
2. 정지 후: 승객 서서히 직립 복원, "정지" 라벨

### 추가 관성 실험
- "동전 실험" 버튼:
  - 쌓인 동전 5개(CylinderGeometry r=0.15, h=0.03, 0xccaa44) 테이블 위에 배치
  - 자(BoxGeometry 가느다란 막대)로 맨 아래 동전을 빠르게 치는 애니메이션
  - 맨 아래 동전만 날아가고, 위 4개는 제자리에 남음 → 관성 시연
  - "위 동전들은 정지해 있으려는 관성!" 라벨

## 탭 2: 운동량 비교 (교과서 p133~134, 그림 III-23, III-24)

### 4가지 물체 비교 장면
- 레일(BoxGeometry 길쭉한 직사각형, 0x888888) 4줄 나란히 배치
- 각 레일 위에 물체가 이동:

| 물체 | 3D 형상 | 크기 | 색상 | 질량 | 속도 | 운동량 |
|------|---------|------|------|------|------|--------|
| 야구공 | SphereGeometry(r=0.12) | 작음 | 0xff4444 빨강 | 0.15 kg | 40 m/s | 6 kg·m/s |
| 볼링공 | SphereGeometry(r=0.35, MeshStandardMaterial roughness 0.3, metalness 0.2) | 중간 | 0x2244aa 진한 파랑, 3개 손가락 홈(SphereGeometry r=0.05, 0x111111) | 6 kg | 5 m/s | 30 kg·m/s |
| 자전거+사람 | BoxGeometry 조합: 프레임(CylinderGeometry 2개 대각선)+바퀴(TorusGeometry r=0.4, tube=0.03) 2개 + 사람 실루엣 | 큼 | 0x44aa44 초록 | 80 kg | 5 m/s | 400 kg·m/s |
| 고속열차 | BoxGeometry(3, 0.6, 0.5, 둥근 앞부분: SphereGeometry 반구), MeshStandardMaterial(0xff8844, metalness 0.5) | 매우 큼 | 0xff8844 주황 | 300,000 kg | 80 m/s | 24,000,000 kg·m/s |

- 각 물체가 좌→우로 이동, 속도에 비례하는 이동 속력
- 각 물체 위에 라벨: "m=[질량] v=[속도] → p=[운동량]"

### 정지한 고속열차 (교과서 그림 III-24, p133)
- 5번째 레일: 고속열차와 동일 모형이지만 정지 상태
- 라벨: "m=300,000 kg, v=0 → p=0 (운동량 없음!)"
- "질량이 크지만 속도가 0이므로 운동량은 0이다" 강조 라벨 (노란 배경)

### 운동량 비교 막대그래프 (하단 HTML overlay)
- Canvas 2D 렌더링, 로그 스케일 가로 막대:
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
야구공       █░░░░░░░░░░░░░░░  6 kg·m/s
볼링공       ██░░░░░░░░░░░░░░  30 kg·m/s
자전거+사람  █████░░░░░░░░░░░  400 kg·m/s
달리는 열차  ████████████████  24,000,000 kg·m/s
정지한 열차  ░░░░░░░░░░░░░░░░  0 kg·m/s ❌
```
- 각 막대 색상 = 물체 색상
- 호버 시 정확한 값 툴팁

### 운동량 계산기 (인터랙티브)
- "사용자 정의" 모드:
  - "질량" 슬라이더: 0.01 ~ 100,000 (기본값 1), 로그 스케일, 단위 kg
  - "속도" 슬라이더: 0 ~ 100 (기본값 5), 단위 m/s
  - 계산 결과: "운동량 p = m × v = [질량] × [속도] = [운동량] kg·m/s" 실시간 표시
  - 물체 크기가 질량에 비례 (scale 변화)
  - 물체 이동 속도가 슬라이더 속도에 비례

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "실험 탭" 드롭다운: 관성 체험 / 운동량 비교
- 관성 탭:
  - "급출발" 버튼: 출발 애니메이션 재생
  - "급정지" 버튼: 정지 애니메이션 재생 (등속 운동 중에만 활성)
  - "동전 실험" 버튼: 동전 관성 실험 재생
  - "기울어짐 각도" 슬라이더: 5° ~ 30° (기본값 15°) — 승객 기울어짐 정도
  - "버스 속도" 슬라이더: 1 ~ 10 (기본값 5), 단위 m/s
- 운동량 탭:
  - "물체 선택" 드롭다운: 야구공 / 볼링공 / 자전거+사람 / 고속열차 / 정지한 열차 / 사용자 정의
  - "질량" 슬라이더 (사용자 정의 시): 0.01 ~ 100,000, 로그 스케일
  - "속도" 슬라이더 (사용자 정의 시): 0 ~ 100
  - "전체 비교" 토글: ON 시 5개 물체 동시 표시
- 공통:
  - "슬로모션" 슬라이더: 0.2x ~ 3x (기본값 1x), 0.1 단계
  - "관성 화살표 표시" 토글: ON/OFF (기본값 ON)
  - "초기화" 버튼

## 시각화 규칙

### 관성 기울어짐 방향 강조
- 급출발 시: 승객 뒤로 기울어짐 → 기울어진 방향에 반투명 파란 영역(관성 방향 강조)
- 급정지 시: 승객 앞으로 기울어짐 → 빨간 영역
- 기울어짐과 동시에 바닥에 가속도 방향 화살표 표시

### 운동량 크기 시각화
- 각 물체 뒤에 운동량에 비례하는 "꼬리"(반투명 삼각형 BufferGeometry, 길이 = log(p) 비례)
- 운동량이 0인 정지 열차: 꼬리 없음, 회색 반투명(opacity 0.3)으로 표시

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널(배경 rgba(0,0,0,0.7), 둥근 모서리 12px, padding 20px)로 표시:

```
🚌 관성과 운동량
━━━━━━━━━━━━━━

[관성 탭]
관성: 물체가 운동 상태를 유지하려는 성질
  정지 물체 → 외부 힘 없으면 → 계속 정지
  운동 물체 → 외부 힘 없으면 → 계속 등속 운동

버스 속도: [값] m/s
승객 기울어짐: [값]°
상태: [정지 / 가속 / 등속 / 감속]

[운동량 탭]
운동량 = 질량 × 속도
p = m × v

현재 물체: [이름]
질량:  m = [값] kg
속도:  v = [값] m/s
운동량: p = m × v = [값] kg·m/s

운동량 방향 = 속도 방향
정지 시(v=0) → p = 0
```
모든 값은 실시간 업데이트.

## 교육적 요소
화면 상단에 토글 가능한 "📖 학습 가이드" 패널:
- 핵심 원리: "외부 힘이 없으면 물체는 현재 운동 상태를 유지합니다(관성). 운동량은 질량과 속도의 곱으로, 질량이 크고 빠를수록 운동 상태를 변화시키기 어렵습니다."
- 탐구 질문:
  1. "고속 열차에 안전띠가 없는 까닭은 무엇인가요?" (교과서 p132: 급격한 속도 변화가 적으므로)
  2. "정지한 고속 열차의 운동량은 얼마인가요?" (답: 0 — 질량이 아무리 커도 속도가 0이면 운동량 0)
  3. "야구공과 볼링공이 같은 속도로 날아올 때 어느 것을 멈추기 더 어려운가요? 왜?" (답: 볼링공, 질량이 커서 운동량이 큼)
  4. "피겨 스케이팅 선수가 얼음을 계속 밀지 않아도 미끄러지는 까닭은?" (답: 관성)
  5. "버스에서 넘어지지 않으려면 어떻게 해야 하나요?" (답: 손잡이 잡기 — 외부 힘으로 관성 상쇄)
- 실생활 예시: 안전벨트(관성 대비), 에어백(관성에 의한 충격 완화), 물 미끄럼틀(질량↑ → 운동량↑ → 더 빠르게 미끄러짐), 우주 유영(관성으로 등속 운동 유지)
- 교과서 연계: "미래엔 통합과학1 III편 p132~134 — 관성(그림 III-22), 운동량(그림 III-23 볼링공, 그림 III-24 고속 열차/물 미끄럼틀)"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원 (OrbitControls 기본 지원), 모바일에서 lil-gui 기본 접힘(collapsed) 상태 시작
- 버스 이동: position.x += velocity * dt (requestAnimationFrame, Clock.getDelta())
- 승객 기울어짐: 상체 Group의 rotation.z를 lerp로 목표 각도까지 보간 (0.3초)
- 도로 중앙선: position.x -= velocity * dt (역방향 이동, 범위 밖 시 리셋 = 무한 도로 효과)
- 바퀴 회전: rotation.z -= velocity * dt / wheelRadius
- 운동량 막대: Canvas 2D API (별도 <canvas>), 로그 스케일 = Math.log10(p) 비례 길이
- 탭 전환 시 이전 씬 객체 dispose() + 새 씬 구성
- 성능: requestAnimationFrame 루프, 승객 3명 + 좌석 4개 + 바퀴 4개 = 총 ~30개 mesh
- 파일명: inertia_momentum_sim.html

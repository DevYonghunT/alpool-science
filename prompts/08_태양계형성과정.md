너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 통합과학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 통합과학1
- 단원: II. 물질과 규칙성 > 1. 원소의 생성과 규칙성 > 별의 진화와 원소의 생성
- 토픽: 태양계 형성 과정
- 대상: 고등학교 1학년
- 핵심 개념: 약 50억 년 전 태양계 성운이 회전하면서 수축해 원시 태양을 형성했다. 원시 원반에서 미행성체가 형성되고, 미행성체가 충돌하면서 성장해 행성이 되었다. 태양 가까이에는 암석과 금속의 지구형 행성이, 먼 곳에는 기체의 목성형 행성이 형성되었다.

## 씬 구성
- 전체 화면 3D 우주 공간
- 중앙에 태양계 형성 과정을 3D로 렌더링한다.
- 배경: 우주 0x000008, 별 파티클 Points(1500개)
- 카메라: PerspectiveCamera(FOV 60) + OrbitControls
  - 기본 시점: 약간 위에서 비스듬히 내려다보는 각도 (태양계 원반을 잘 볼 수 있게)
- 조명: AmbientLight(0.1) + PointLight(중심, 세기 단계별 변화)

## 형성 과정 5단계 상세 (교과서 그림 II-12, p57)

### 단계 1: 태양계 성운의 형성과 수축
- 3D: 거대한 가스 구름
  - Points(2000개) 구형 분포, 반지름 ~15
  - 파티클 색상: 0x4455aa(파랑) + 0x6644aa(보라) + 0x224466(청록) 랜덤
  - 파티클 크기: 0.3~1.5 랜덤
  - 반투명 구체 SphereGeometry(r=15, opacity 0.04) 여러 겹 (성운 구름 효과)
- 수축 애니메이션:
  1. 성운 전체가 천천히 회전 시작 (회전 속도: 0 → 0.005 rad/frame, 5초)
  2. 파티클이 중심으로 서서히 수축 (반지름 15 → 8, 5초)
  3. 회전이 빨라지면서 구형 → 납작한 원반 형태로 변형
     - Y축 방향 파티클 위치가 서서히 0에 수렴 (구 → 디스크)
  4. 중심부가 밝아짐 (PointLight 세기 0.5 → 3.0)
- 교과서: "밀도가 큰 부분을 중심으로 회전하면서 서서히 수축했다" (p57 그림 II-12)

### 단계 2: 원시 태양 형성
- 3D: 중심에 밝은 원시 태양 + 주변 원반
  - 원시 태양: SphereGeometry(r=1.5), MeshStandardMaterial(emissive 0xff8800, emissiveIntensity 0.8)
  - 원시 원반: RingGeometry(innerRadius=2, outerRadius=12, thetaSegments=128)
    - 재질: MeshStandardMaterial(0x997744, opacity 0.5, side: DoubleSide)
    - 원반 위에 먼지/가스 파티클(Points 500개) 원반 면을 따라 분포, 천천히 공전
  - 원반 내부에 밀도 차이: 중심 가까이 밝고, 바깥쪽 어두움
- 교과서: "성운의 회전 속도가 빨라지면서 중심부에 원시 태양, 주변부에 납작한 원시 원반이 만들어졌다" (p57)

### 단계 3: 미행성체 형성
- 3D: 원시 원반 위에 작은 암석 덩어리들 생성
  - 미행성체: IcosahedronGeometry(r=0.1~0.3, detail=1) 불규칙 형태, 50개
  - 색상: 0x887766(암석 갈색) + 0xaaaaaa(금속 회색) 랜덤
  - 원반을 따라 공전 (각각 다른 궤도 반지름, 다른 공전 속도)
  - 공전 속도: 케플러 법칙 근사 — 안쪽 궤도일수록 빠르게
- 형성 애니메이션:
  - 원반의 먼지 파티클 일부가 모여들어 미행성체로 응집하는 효과 (파티클 수 줄어들며 미행성체 생성)
- 교과서: "원시 원반에서 티끌 같은 고체 물질들이 뭉쳐 미행성체가 만들어졌다. 크기는 보통 수 km 정도이다." (p57)

### 단계 4: 원시 행성 형성 — 미행성체 충돌
- 3D: 미행성체끼리 충돌하며 커지는 과정
  - 충돌 애니메이션:
    1. 두 미행성체가 서로에게 접근 (궤도 교차)
    2. 충돌 순간: 밝은 플래시(PointLight 순간 세기 5.0) + 파편 파티클(Points 20개, 방사형)
    3. 충돌 후: 더 큰 하나의 천체로 합쳐짐 (반지름 증가)
  - 반복하여 50개 미행성체 → 8개 원시 행성으로 축소
  - 충돌 횟수 카운터 표시
- 구분 표시:
  - 태양 가까이(궤도 반지름 2~5): 암석+금속 → 지구형 행성 (갈색/회색)
  - 태양 멀리(궤도 반지름 6~12): 가스+얼음 → 목성형 행성 (연한 주황/파랑)
  - 각 원시 행성에 "지구형" / "목성형" 라벨
- 교과서: "미행성체가 충돌하면서 성장해 원시 행성을 형성했다" (p57)

### 단계 5: 현재의 태양계
- 3D: 완성된 태양계
  - 태양: SphereGeometry(r=1.2), emissive 0xffcc00, emissiveIntensity 1.0, 코로나 효과(반투명 구 r=1.5, 0xffdd44, opacity 0.15)
  - 8개 행성 (태양에서 가까운 순):
    | 행성 | 궤도 반지름 | 구 반지름 | 색상 | 유형 |
    |------|-----------|----------|------|------|
    | 수성 | 2.0 | 0.08 | 0x999999 회색 | 지구형 |
    | 금성 | 2.8 | 0.15 | 0xddaa66 노란갈색 | 지구형 |
    | 지구 | 3.5 | 0.16 | 0x2266aa 파랑+녹색 | 지구형 |
    | 화성 | 4.2 | 0.10 | 0xcc4422 붉은색 | 지구형 |
    | 목성 | 6.5 | 0.60 | 0xdd9944 주황줄무늬 | 목성형 |
    | 토성 | 8.0 | 0.50 | 0xccaa77 연한 노랑 | 목성형 |
    | 천왕성 | 9.5 | 0.30 | 0x88ccdd 연한 청록 | 목성형 |
    | 해왕성 | 11.0 | 0.28 | 0x3355aa 진한 파랑 | 목성형 |
  - 토성 고리: RingGeometry(innerRadius=0.65, outerRadius=0.95), 0xddcc99, opacity 0.6
  - 궤도 표시: 각 행성 궤도를 얇은 원(RingGeometry 또는 Line)으로 표시, 0x333333 연한 회색
  - 모든 행성이 각자의 속도로 공전 (안쪽 행성 빠르게, 바깥 느리게)
- 구분선: 화성과 목성 사이에 점선 원 + "← 지구형 행성 | 목성형 행성 →" 라벨
- 교과서: "태양과 가까운 곳에 암석 성분의 지구형 행성이, 먼 곳에 기체 성분의 목성형 행성이 존재한다" (p57)

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "형성 단계" 슬라이더: 0 ~ 4 (5단계, 정수)
  - 0: 성운 수축 / 1: 원시 태양+원반 / 2: 미행성체 형성 / 3: 충돌+행성 형성 / 4: 현재 태양계
- "자동 재생" 버튼: 단계 0→4 자동 진행 (각 단계 6초, 전환 3초)
- "공전 속도" 슬라이더: 0x ~ 10x (기본값 1x), 0.5 단계 (단계 4에서만 활성)
- "행성 이름 표시" 토글: ON/OFF (기본값 ON)
- "궤도 표시" 토글: ON/OFF (기본값 ON)
- "시점" 드롭다운: 위에서 보기(기본) / 옆에서 보기 / 지구 시점
- "초기화" 버튼: 단계 0, 기본 시점

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널로 표시:

```
☀️ 태양계 형성 과정
━━━━━━━━━━━━━━━━
현재 단계:     [단계명]
경과 시간:     약 [값]억 년 전
태양 나이:     약 50억 년

태양계 구분:
  지구형 행성: 수성, 금성, 지구, 화성
    → 암석 + 금속, 작고 밀도 높음
  목성형 행성: 목성, 토성, 천왕성, 해왕성
    → 수소 + 헬륨(기체), 크고 밀도 낮음

형성 원리:
  태양 가까이 → 고온 → 기체 날아감 → 암석+금속만 남음
  태양 멀리 → 저온 → 기체 유지 → 거대 기체 행성
```

  1. "태양계 성운이 회전하면서 수축할 때 왜 납작한 원반 모양이 되었을까요?"
  2. "태양에서 가까운 곳에 지구형 행성이, 먼 곳에 목성형 행성이 형성된 까닭은 무엇인가요?"
  3. "미행성체가 충돌하여 행성이 된다면, 현재도 소행성 충돌이 일어날 수 있을까요?"
- 실생활 예시: NASA 태양계 탐사 (보이저, 뉴 허라이즌스), 소행성 방어 (DART 미션), 외계 행성 탐색 (케플러, TESS)
- 교과서 연계: "미래엔 통합과학1 II편 p57 — 태양계의 형성 과정 (그림 II-12)"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원 (OrbitControls 기본 지원)
- 성운 파티클: BufferGeometry + Points (2000개)
- 행성 공전: requestAnimationFrame에서 각도 업데이트 (속도 = 1/√r 근사)
- 충돌 애니메이션: 타이머 기반 시퀀스
- 단계 전환 시 이전 객체 dispose()
- 성능: requestAnimationFrame 루프, InstancedMesh로 미행성체 렌더링
- 파일명: solar_system_formation_sim.html

너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 물리학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 역학과 에너지
- 단원: I. 시공간과 운동 > 05. 역학적 에너지 보존과 탈출 속도
- 토픽: 탈출 속도와 블랙홀
- 대상: 고등학교 (진로 선택과목)
- 핵심 개념: 탈출 속도는 천체의 중력을 벗어나 무한히 멀어지는 데 필요한 최소 속도로, v_esc = √(2GM/R)이다. 지구의 탈출 속도는 약 11.2km/s이다. 천체의 질량이 크거나 반지름이 작을수록 탈출 속도가 커진다. 탈출 속도가 빛의 속력 c보다 큰 천체에서는 빛조차 빠져나올 수 없는데, 이를 블랙홀이라 한다. 블랙홀의 경계를 사건의 지평선이라 하며, 슈바르츠실트 반지름 r_s = 2GM/c²이다.

## 씬 구성
- 화면 좌측 60%: 3D 천체 + 발사체 궤적 / 블랙홀 시각화
- 화면 우측 40%: 탈출 속도 비교 표 + 에너지 다이어그램
- 2가지 모드: 행성별 탈출 속도 비교 / 블랙홀
- 배경: 0x000008 (깊은 우주)
- 카메라: PerspectiveCamera(FOV 50) + OrbitControls
- 조명: AmbientLight(0.1) + PointLight(0.5)

## 모드 1: 행성별 탈출 속도 비교

### 3D
- 천체 선택: 지구/달/화성/목성/태양
  - 지구: SphereGeometry(r=2), 0x2266aa, v_esc ≈ 11.2km/s
  - 달: SphereGeometry(r=0.8), 0xcccccc, v_esc ≈ 2.4km/s
  - 화성: SphereGeometry(r=1.5), 0xcc5533, v_esc ≈ 5.0km/s
  - 목성: SphereGeometry(r=4), 0xddaa66, v_esc ≈ 59.5km/s
  - 태양: SphereGeometry(r=5), emissive 0xffcc00, v_esc ≈ 617.5km/s

### 발사 시뮬레이션
- 발사 속도 v₀ 슬라이더: 0 ~ 1.5×v_esc
- v₀ < v_esc: 발사체가 올라갔다 다시 떨어짐 (빨간 궤적)
  - 최대 높이 h_max = v₀²/(2g) (지표면 근처 근사)
- v₀ = v_esc: 발사체가 무한히 멀어짐, 속도→0 (초록 궤적)
- v₀ > v_esc: 발사체가 무한히 멀어지면서 양의 속도 유지 (파란 궤적)
- 궤적과 속도 변화를 3D + 그래프로 동시 표시

### 탈출 속도 비교 막대그래프 (우측)
- 5개 천체의 v_esc 비교
- 막대 높이 = v_esc에 비례
- 현재 선택된 천체 강조

## 모드 2: 블랙홀

### 3D 블랙홀 시각화
- 블랙홀 중심: SphereGeometry(r=1), 0x000000 (완전 검정)
- 사건의 지평선(event horizon): SphereGeometry(r=r_s), 0x111122 반투명
  - r_s = 2GM/c² (슈바르츠실트 반지름)
  - 태양 질량 블랙홀: r_s ≈ 3km
- 강착 원반: RingGeometry(r_s, 3r_s), 발광 효과
  - 색상 그라데이션: 안쪽 0xff4400(뜨거운 주황) → 바깥 0xff0044(빨강)
  - 회전 애니메이션
- 빛의 휨(중력 렌즈): 블랙홀 뒤쪽 별빛이 휘어지는 효과
  - 배경 별들의 위치가 블랙홀 근처에서 왜곡

### 탈출 속도 vs 빛의 속력
- 슬라이더로 천체의 질량 M 또는 반지름 R 조절
- v_esc = √(2GM/R) 실시간 계산
- v_esc 막대 vs c(빛의 속력 3×10⁸m/s) 막대 비교
- v_esc > c: "⚠️ 블랙홀! 빛조차 탈출 불가" 경고
- v_esc = c 되는 순간의 반지름 = 슈바르츠실트 반지름 r_s

### 빛의 탈출 시각화
- 블랙홀 표면에서 빛(노란 파티클) 발사:
  - r > r_s: 빛이 탈출 (밖으로 퍼져나감)
  - r = r_s: 빛이 원 궤도 (사건의 지평선에서 맴돔)
  - r < r_s: 빛이 블랙홀로 빨려들어감 (빛도 탈출 불가!)

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "모드" 드롭다운: 행성별 탈출 속도 / 블랙홀
- "천체" 드롭다운 (모드 1): 지구 / 달 / 화성 / 목성 / 태양
- "발사 속도" 슬라이더 (모드 1): 0~1.5×v_esc
- "블랙홀 질량" 슬라이더 (모드 2): 1~100 태양 질량 (로그)
- "사건의 지평선 표시" 토글: ON/OFF (기본값 ON)
- "강착 원반 표시" 토글: ON/OFF (기본값 ON)
- "빛 궤적 표시" 토글: ON/OFF
- "발사" 버튼
- "초기화" 버튼


## 교육적 요소

- 핵심 원리: "탈출 속도 v_esc=√(2GM/R)는 천체의 질량이 크거나 반지름이 작을수록 커집니다. 탈출 속도가 빛의 속력보다 큰 천체가 블랙홀이며, 사건의 지평선 안에서는 빛조차 빠져나올 수 없습니다."
- 탐구 질문:
  1. "지구의 탈출 속도 11.2km/s를 역학적 에너지 보존으로 유도해 보세요."
  2. "달의 탈출 속도가 지구보다 작은 까닭을 설명해 보세요."
  3. "태양이 블랙홀이 되려면 반지름이 얼마나 작아져야 하나요? (힌트: r_s = 2GM/c²)"
- 실생활 예시: 로켓 발사 속도, 목성형 행성의 대기 유지, 블랙홀 관측(EHT, M87*)
- 교과서 연계: "역학과 에너지 I편 p42~51 — 탈출 속도, 블랙홀"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원 (OrbitControls 기본 지원)

- gui.controllersRecursive().forEach(c => c.updateDisplay()) 동기화
- 성능: requestAnimationFrame 루프, 파티클 최대 200개
- 파일명: mech_10_escape_velocity_blackhole.html

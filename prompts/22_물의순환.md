너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 통합과학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 통합과학1
- 단원: III. 시스템과 상호작용 > 1. 지구시스템 > 지구시스템의 구성과 상호작용
- 토픽: 물의 순환
- 대상: 고등학교 1학년
- 핵심 개념: 물의 순환은 태양 에너지에 의해 일어난다. 수권의 물이 태양 에너지를 흡수하면 수증기가 되어 기권으로 이동한다. 기권으로 이동한 수증기는 에너지를 방출하면서 응결해 구름이 되었다가 비나 눈의 형태로 지표로 이동한다. 지표로 이동한 물은 지하로 스며들거나 지표를 따라 낮은 곳으로 흐르면서 지형을 변화시킨다. 이 과정에서 물질의 순환과 에너지의 흐름이 함께 일어난다.

## 씬 구성
- 전체 화면에 3D 풍경(지형+바다+산+하천)을 렌더링한다.
- 풍경 위에서 물의 순환 사이클이 애니메이션으로 반복된다.
- 배경: 하늘 그라데이션 (0x87CEEB 위 → 0xE0F0FF 아래)
- 카메라: PerspectiveCamera(FOV 55) + OrbitControls
- 조명: AmbientLight(0.4) + DirectionalLight(태양, 0.7, 좌상단)

## 3D 풍경 구성
- 지형: PlaneGeometry(20, 15, 50, 50), Y축 높이를 Perlin noise로 변형하여 산과 골짜기 생성
  - 산: 좌측 높은 영역 (최고 높이 5.0)
  - 골짜기/하천: 산에서 바다로 이어지는 낮은 골짜기 경로
  - 평야: 중간 높이 영역
  - 해안: 우측 낮은 영역 (높이 0.0 이하 = 바다)
- 지형 텍스처 (높이에 따라):
  - 산 꼭대기(>4.0): 0xffffff 흰색 (눈)
  - 산 중턱(2.5~4.0): 0x887766 암석 갈색
  - 평야(0.5~2.5): 0x44aa44 풀밭 녹색
  - 해안(0~0.5): 0xddcc88 모래
- 바다: PlaneGeometry(10, 15), 0x2266aa, opacity 0.7, Y=0.0, 물결 sin 애니메이션
- 하천: TubeGeometry(곡선 경로, r=0.1), 0x4488cc, 산에서 바다로 이어짐
- 지하수: 지형 아래에 반투명 파란 영역(BoxGeometry, 0x224488, opacity 0.2)
- 태양: SphereGeometry(r=1.0), emissive 0xffcc00, 좌측 상단 배치

## 물의 순환 사이클 애니메이션

### 1단계: 증발 (바다 → 기권)
- 바다/하천 표면에서 작은 파란 파티클(Points, r=0.03, 0x88ccff)이 위로 올라감
- 파티클이 올라가면서 점점 투명해짐 (수증기화)
- 바다 표면에 열 이펙트: 미세한 아지랑이 효과 (ShaderMaterial distortion)
- 태양에서 바다로 향하는 노란 화살표: "태양 에너지"
- 라벨: "증발 — 태양 에너지를 흡수하여 수증기로 변함"
- 화살표 방향: ↑ (위로)

### 2단계: 응결 + 구름 형성 (기권)
- 올라간 수증기 파티클이 특정 높이(Y=6.0)에 도달하면 구름으로 변환
- 구름: SphereGeometry 여러 개(r=0.5~1.0) 뭉친 형태, 0xffffff, opacity 0.7
- 구름 3~5개가 산 위에서 바다 방향으로 천천히 이동 (바람에 의해)
- 응결 시 에너지 방출: 구름 주위 노란 글로우(emissive 0.1) 깜빡임
- 라벨: "응결 — 수증기가 에너지를 방출하며 구름 형성"

### 3단계: 강수 (기권 → 지표)
- 구름 아래에서 비 파티클(Points, 100개, 0x4488ff) 내려옴
- 비 파티클: 가느다란 수직 막대 형태 (BoxGeometry h=0.15, w=0.01)
- 산 위에서 눈 파티클(Points, 30개, 0xffffff): 작은 점, 천천히 내려옴
- 라벨: "강수 — 비나 눈의 형태로 지표로 이동"

### 4단계: 지표 흐름 + 침투 (지표)
- 비가 지표에 닿으면:
  - 일부: 하천을 따라 산→바다로 흐르는 파란 파티클(하천 경로를 따라 이동)
  - 일부: 지하로 스며드는 파란 파티클(아래로 이동, 지하수 영역으로)
- 하천 흐름: TubeGeometry 위에 파티클이 빠르게 이동
- 지하수 흐름: 지형 아래에서 느리게 수평 이동
- 라벨: "지표 흐름 — 낮은 곳으로 흐르며 지형 변화"

### 5단계: 바다로 복귀
- 하천을 통해 바다에 도착한 파티클이 바다에 합류
- 지하수를 통해 해안에서 바다로 스며나오는 파티클
- 순환 완료 → 1단계로 반복

### 전체 사이클 동시 진행
- 5단계가 동시에 진행: 증발+응결+강수+흐름+복귀 모두 같은 시간에 진행
- 총 파티클: 약 200~300개, 각각 사이클의 다른 위치에 분포

## 지형 변화 시각화 (교과서 p109)
- "지형 변화 모드" 토글 ON 시:
  - V자 계곡: 산 상류에 좁은 골짜기 형태 강조
  - 선상지: 산 아래 부채꼴 모양 퇴적 지형
  - 곡류: 평야에서 구불구불한 하천
  - 삼각주: 하천이 바다로 만나는 지점에 삼각형 퇴적
  - 각 지형에 이름 라벨
- 교과서: "V자 모양의 계곡이나 부채꼴 모양의 선상지, 구불구불하게 흐르는 곡류, 모래나 점토가 쌓여 만들어진 삼각주 등은 물의 순환 과정에서 형성된 지형이다." (p109)

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "순환 속도" 슬라이더: 0.5x ~ 5x (기본값 1x)
- "태양 에너지 강도" 슬라이더: 0.2 ~ 2.0 (기본값 1.0)
  - 강도↑ → 증발량↑ → 구름 많아짐 → 강수량↑
  - 강도↓ → 증발량↓ → 구름 적어짐 → 강수량↓
- "단계별 보기" 드롭다운: 전체 / 증발만 / 응결만 / 강수만 / 지표 흐름만
- "지형 변화 모드" 토글: ON/OFF (기본값 OFF)
- "에너지 흐름 표시" 토글: ON/OFF (기본값 ON)
  - 태양→바다 에너지, 증발 시 흡수, 응결 시 방출 화살표
- "권역 라벨 표시" 토글: ON/OFF (기본값 ON)
  - 기권, 수권, 지권 영역 라벨
- "초기화" 버튼

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널로 표시:

```
💧 물의 순환
━━━━━━━━━━
순환 동력:     태양 에너지
현재 태양 강도: [값]x

순환 과정:
  바다(수권) →[증발]→ 수증기(기권)
  →[응결]→ 구름 →[강수]→ 비/눈(지표)
  →[지표 흐름]→ 하천/지하수
  →[복귀]→ 바다(수권)

에너지 흐름:
  증발 시: 태양 에너지 흡수 (잠열)
  응결 시: 에너지 방출 (잠열)

물의 순환 → 물질의 순환 + 에너지의 흐름
```

## 교육적 요소

- 탐구 질문:
  1. "물의 순환을 일으키는 지구시스템의 에너지원은 무엇인가요?" (답: 태양 에너지)
  2. "물이 순환하면서 지표에 다양한 지형이 만들어지는 과정을 설명해 보세요."
  3. "만약 태양 에너지가 줄어든다면 물의 순환에 어떤 변화가 일어날까요?"
- 실생활 예시: 댐(하천 물 저장), 지하수 우물, 수력 발전(물의 위치 에너지), 기상 예보(강수 예측)
- 교과서 연계: "미래엔 통합과학1 III편 p108~109 — 물의 순환과 에너지 흐름, 물의 순환과 지형"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js CDN: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어, 반응형, 모바일 터치 지원
- 지형: PlaneGeometry + vertex Y displacement (Perlin noise 또는 simplex)
- 물 파티클: BufferGeometry + Points, position attribute 애니메이션
- 구름: SphereGeometry 여러 개 Group으로 묶어 이동
- 비/눈: Points, Y축 감소
- 하천 경로: CatmullRomCurve3, 파티클이 curve.getPointAt(t)로 이동
- 성능: requestAnimationFrame 루프, 파티클 최대 300개
- 파일명: water_cycle_sim.html

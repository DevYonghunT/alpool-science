너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 물리학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 전자기와 양자
- 단원: III. 양자와 미시세계 > 01. 양자의 기묘한 세계
- 토픽: 불확정성 원리 (하이젠베르크)
- 대상: 고등학교 (진로 선택과목)
- 핵심 개념: 하이젠베르크의 불확정성 원리에 따르면 입자의 위치와 운동량을 동시에 정확하게 측정할 수 없다. ΔxΔp ≥ ℏ/2 (ℏ = h/(2π)). 위치를 정확히 측정하면 운동량의 불확정성이 커지고, 운동량을 정확히 측정하면 위치의 불확정성이 커진다. 이는 측정 기술의 한계가 아니라 양자 세계의 본질적 성질이다.

## 씬 구성
- 화면 좌측: 3D 전자 위치/운동량 불확정성 시각화
- 화면 우측: Δx-Δp 관계 그래프 (Canvas 2D)
- 2가지 시나리오: 단일 슬릿 전자 회절 / 전자 구름 비유
- 배경: 0x0a0a20
- 카메라: PerspectiveCamera(FOV 50) + OrbitControls
- 조명: AmbientLight(0.3)

## 시나리오 1: 단일 슬릿 전자 회절로 불확정성 원리 이해

### 3D
- 전자 빔: 왼쪽에서 오른쪽으로 전자(파란 파티클) 발사
- 슬릿: BoxGeometry(0.05, 4, 3), 0x333333, 슬릿 폭 Δx 조절 가능
- 스크린: PlaneGeometry, 오른쪽에 배치

### 불확정성 시각화
- **슬릿 폭 Δx를 줄이면** (위치를 더 정확히 아는 것):
  - 전자의 위치 불확정성 Δx↓
  - 하지만 회절에 의해 전자가 더 넓게 퍼짐 → 운동량(y방향) 불확정성 Δp_y↑
  - 스크린의 패턴이 넓어짐
  - "위치를 정확히 알수록 운동량을 더 모르게 된다"

- **슬릿 폭 Δx를 키우면** (위치를 대략 아는 것):
  - Δx↑, 하지만 회절 약함 → Δp_y↓
  - 스크린의 패턴이 좁아짐
  - "위치를 대략 알면 운동량은 비교적 정확"

- ΔxΔp ≥ ℏ/2 관계를 슬릿 폭 슬라이더로 직접 체험

### 전자 하나씩 발사 모드
- 전자를 1개씩 발사 → 스크린에 점 하나씩 찍힘 (랜덤 위치)
- 전자 수↑: 점이 쌓이면서 회절 무늬 패턴 형성
- "전자 1개도 확률적으로 회절 무늬를 따른다" → 파동-입자 이중성

## 시나리오 2: 전자 구름 (위치 불확정성)

### 3D
- 원자핵: SphereGeometry(r=0.2), 0xff4444
- 전자의 "위치": 확률 분포로만 표현 가능
  - Δx 작게 설정: 전자가 좁은 영역에 밀집 → 빠르게 진동(Δp 큼)
  - Δx 크게 설정: 전자가 넓은 영역에 퍼짐 → 느리게 진동(Δp 작음)
- 전자 구름: Points(500개), 확률 밀도에 따라 분포
  - 중심에서 밀도 높고, 바깥에서 낮음 (가우시안)
  - Δx 슬라이더 → 구름 폭 변화 + 운동량 분포 변화

## Δx-Δp 관계 그래프 (우측 Canvas 2D)
- X축: Δx (위치 불확정성)
- Y축: Δp (운동량 불확정성)
- 곡선: ΔxΔp = ℏ/2 (최소 불확정성, 쌍곡선)
- 허용 영역: 쌍곡선 위쪽 (ΔxΔp ≥ ℏ/2)
- 금지 영역: 쌍곡선 아래 (불가능)
- 현재 (Δx, Δp) 위치에 빨간 마커
- "Δx↓ → Δp↑, Δx↑ → Δp↓ (반비례 관계)"

## 고전 vs 양자 비교
| | 고전 역학 | 양자 역학 |
|---|---------|---------|
| 위치 | 정확히 결정 | 확률 분포 |
| 운동량 | 정확히 결정 | 확률 분포 |
| 동시 측정 | 가능 | 불가능 (ΔxΔp≥ℏ/2) |
| 궤적 | 결정론적 | 확률적 |

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "시나리오" 드롭다운: 슬릿 회절 / 전자 구름
- "슬릿 폭 Δx" 슬라이더: 0.1~5 (기본값 1)
- "전자 수" 슬라이더 (슬릿): 10~1000 (기본값 100)
- "전자 1개씩 발사" 토글: ON/OFF
- "Δx-Δp 그래프" 토글: ON/OFF (기본값 ON)
- "비교 표" 토글: ON/OFF
- "전자 발사" 버튼
- "초기화" 버튼

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널로 표시:

```
🔮 불확정성 원리 (하이젠베르크)
━━━━━━━━━━━━━━━━━
시나리오: [슬릿/전자구름]

하이젠베르크 불확정성 원리:
  ΔxΔp ≥ ℏ/2
  ℏ = h/(2π) = 1.055×10⁻³⁴ J·s

현재:
  Δx = [값]m (위치 불확정성)
  Δp = [값]kg·m/s (운동량 불확정성)
  ΔxΔp = [값] ≥ ℏ/2 ✅

의미:
  위치를 정확히 알수록 → 운동량을 모름
  운동량을 정확히 알수록 → 위치를 모름
  이것은 측정 한계가 아닌 자연의 본질!
```

## 교육적 요소
화면 상단에 토글 가능한 "📖 학습 가이드" 패널:
- 핵심 원리: "하이젠베르크의 불확정성 원리에 따르면 입자의 위치와 운동량을 동시에 정확하게 측정할 수 없습니다. ΔxΔp≥ℏ/2. 이는 양자 세계의 본질적 성질입니다."
- 탐구 질문:
  1. "슬릿 폭을 줄이면 전자의 회절 무늬가 넓어지는 까닭을 불확정성 원리로 설명해 보세요."
  2. "일상적인 물체(야구공)에서 불확정성 원리가 관측되지 않는 까닭은? (힌트: ℏ가 매우 작음)"
  3. "전자의 궤도라는 개념 대신 전자 구름(확률 분포)을 사용하는 까닭은?"
- 실생활 예시: 전자 현미경(분해능 한계), 양자 암호(측정하면 상태 변화), STM(터널링)
- 교과서 연계: "전자기와 양자 III편 p98~103 — 양자의 기묘한 세계, 불확정성 원리"

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원 (OrbitControls 기본 지원)
- 시나리오 전환 시 이전 객체 dispose() + 새 객체 생성
- gui.controllersRecursive().forEach(c => c.updateDisplay()) 동기화
- 성능: requestAnimationFrame 루프, 전자 파티클 InstancedMesh 최대 1000개
- 파일명: emq_21_uncertainty_principle.html

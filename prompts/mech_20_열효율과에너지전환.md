너는 Three.js 과학 시뮬레이션 전문 개발자다. 고등학교 물리학 교과서 기반의 인터랙티브 3D 시뮬레이션을 단일 HTML 파일로 제작한다.

## 시뮬레이션 정보
- 과목: 역학과 에너지
- 단원: II. 열과 에너지 > 04. 열기관과 열효율
- 토픽: 열효율과 에너지 전환
- 대상: 고등학교 (진로 선택과목)
- 핵심 개념: 열효율 η = W/Q_H = (Q_H - Q_L)/Q_H는 열기관이 공급받은 열에너지를 역학적 에너지(일)로 전환하는 비율이다. 실제 열기관의 효율은 카르노 효율보다 항상 낮다. 가솔린 엔진(~25%), 디젤 엔진(~35%), 화력 발전소(~40%). 열효율을 높이려면 고온 열원 온도를 높이거나 저온 열원 온도를 낮춰야 한다.

## 씬 구성
- 화면 중앙: 3D 실제 열기관 모형 (가솔린/디젤 엔진 4행정) + Sankey 에너지 흐름
- 화면 우측: 열효율 비교 막대그래프 (Canvas 2D)
- 배경: 0xeef0f5
- 카메라: PerspectiveCamera(FOV 50) + OrbitControls
- 조명: AmbientLight(0.5) + DirectionalLight(0.6)

## 3D 가솔린 엔진 (4행정)

### 실린더 내부
- 실린더: CylinderGeometry(0.6, 0.6, 2), 투명 0xaaaaaa
- 피스톤: CylinderGeometry(0.58, 0.58, 0.3), 0x666688
- 밸브: CylinderGeometry(0.08, 0.08, 0.3) × 2, 흡기(파랑)/배기(빨강)
- 점화 플러그: CylinderGeometry(0.04, 0.04, 0.2) + 스파크

### 4행정 사이클 + P-V 다이어그램
1. **흡입**: 피스톤↓, 흡기 밸브 열림, 혼합물(파란 파티클) 유입
   - P-V: 낮은 압력, 부피 증가
2. **압축**: 밸브 닫힘, 피스톤↑, 압축 (온도↑)
   - P-V: 단열 압축 곡선
3. **폭발(팽창)**: 점화 스파크 → 폭발(빨간 플래시) → 피스톤↓ (동력!)
   - P-V: 고압에서 팽창 (일 = 면적)
4. **배기**: 배기 밸브 열림, 피스톤↑, 배기 가스(회색 파티클) 배출
   - P-V: 원래 상태로 복귀
- 각 행정 라벨 표시, 자동 순환

### 에너지 흐름 (Sankey)
- Q_H (연료 화학 에너지, 100%): 넓은 빨간 막대
- W (유용한 일, ~25%): 초록 막대
- Q_L (폐열, ~75%): 주황 막대 (냉각수+배기가스+마찰)
- 세부 손실: 냉각수 열(30%), 배기 열(30%), 마찰(15%)

## 디젤 엔진 비교
- 가솔린과 다른 점: 점화 플러그 없음 → 단열 압축으로 자연 발화
- 압축비 더 높음 → 효율 더 높음 (~35%)
- 3D에서 차이점 시각화

## 열효율 비교 막대그래프 (우측)
| 열기관 | 효율 η | 카르노 η_C |
|--------|-------|----------|
| 증기 기관 | 10% | 40% |
| 가솔린 | 25% | 55% |
| 디젤 | 35% | 65% |
| 화력 발전 | 40% | 60% |
| 열병합 | 80%* | — |
(*열병합: 폐열도 난방에 활용 → 총 효율 높음)

### 효율 향상 방법
- T_H 높이기: 고온 연소 → 특수 소재 필요
- T_L 낮추기: 냉각 효율 개선
- 마찰 줄이기: 윤활유, 정밀 가공
- 폐열 회수: 열병합 발전, 배기 열 회수

## 인터랙션 (GUI 패널)
lil-gui 라이브러리(CDN)로 우측 상단에 컨트롤 패널을 배치한다:
- "엔진 종류" 드롭다운: 가솔린 / 디젤 / 증기 기관 / 화력 발전
- "행정 선택" 드롭다운: 흡입 / 압축 / 폭발 / 배기 / 자동 순환
- "Q_H (연료 에너지)" 슬라이더: 100~1000J (기본값 500)
- "재생 속도" 슬라이더: 0.5x~3x
- "Sankey 다이어그램" 토글: ON/OFF (기본값 ON)
- "P-V 다이어그램" 토글: ON/OFF (기본값 ON)
- "효율 비교" 토글: ON/OFF
- "초기화" 버튼

## 수식 패널
화면 좌측 하단에 반투명 HTML overlay 패널로 표시:

```
⚙️ 열효율과 에너지 전환
━━━━━━━━━━━━━━━━
엔진: [가솔린/디젤/증기/화력]
현재 행정: [흡입/압축/폭발/배기]

Q_H = [값]J (연료 에너지)
W = [값]J (유용한 일)
Q_L = [값]J (폐열)

열효율: η = W/Q_H = [값]%
카르노 효율: η_C = 1-T_L/T_H = [값]%
실제 < 카르노 (항상)

효율 향상:
  T_H↑, T_L↓, 마찰↓, 폐열 회수
```

## 교육적 요소
화면 상단에 토글 가능한 "📖 학습 가이드" 패널:
- 핵심 원리: "열효율 η=W/Q_H는 열기관이 공급받은 열을 일로 전환하는 비율입니다. 실제 효율은 카르노 효율보다 항상 낮으며, 폐열 회수 등으로 총 에너지 효율을 높일 수 있습니다."
- 탐구 질문:

  2. "디젤 엔진이 가솔린 엔진보다 효율이 높은 까닭을 압축비와 관련지어 설명해 보세요."
  3. "열병합 발전의 총 효율이 80%나 되는 까닭은 무엇인가요?"
- 실생활 예시: 자동차 연비, 하이브리드/전기차, 열병합 발전, 폐열 회수 시스템

## 코드 규칙
- 단일 HTML 파일로 작성 (외부 파일 없음)
- Three.js는 CDN import: https://unpkg.com/three@0.160.0/build/three.module.js
- OrbitControls CDN: https://unpkg.com/three@0.160.0/examples/jsm/controls/OrbitControls.js
- lil-gui CDN: https://unpkg.com/lil-gui@0.19.2/dist/lil-gui.esm.js
- ES6 모듈 방식 (type="module")
- 주석은 한국어
- 반응형: window resize 이벤트 처리
- 모바일 터치 지원 (OrbitControls 기본 지원)
- 엔진 종류 전환 시 이전 객체 dispose() + 새 객체 생성
- gui.controllersRecursive().forEach(c => c.updateDisplay()) 동기화
- 성능: requestAnimationFrame 루프, 파티클 최대 100개
- 파일명: mech_20_thermal_efficiency.html
